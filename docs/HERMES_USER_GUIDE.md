## 🚀 快速开始

### 前置准备

✅ 已安装 Hermes 语音平台  
✅ 已通过 `bash install-hermes.sh` 安装 Clover A-sales Skill

详见 README.md 中的 [🔑 API Key 配置](../README.md#-api-key-配置) 部分。

## 🎯 核心功能

### 1️⃣ 通话自动复盘（核心特性）

**工作原理**：
- Hermes 通话结束 → 自动启动录音转写
- 转写完成 → 自动调用 Clover Skill 分析
- Clover 分析完成 → 结果自动推送到 Hermes 聊天

**无需用户操作**：
- ❌ 不需要手动配置 Webhook
- ❌ 不需要启动任何服务
- ❌ 不需要访问其他系统或网址
- ✅ Hermes 完成通话 → 自动出现分析结果

**分析内容**：
1. **通话转写**：完整的文字版通话记录
2. **关键信息提取**：
   - 客户名、行业、痛点
   - 销售员提到的解决方案
   - 客户的反对意见和疑虑
   - 约定的下一步行动
3. **十维度评分**：自动给出 10 个维度的能力评分
4. **对比分析**：
   - 与你上次通话的对比
   - 与销冠通话的对比
   - 与团队平均水平的对比
5. **问题诊断**：AI 识别出这通电话的 2-3 个主要问题
6. **改进建议**：具体的话术、方法、练习建议
7. **GROW 教练**：本周的改进计划

### 2️⃣ 十维度动态评分

Clover 会根据每通电话自动评估 10 个维度，体现销售的核心能力。

**特点**：
- **个性化权重**：根据你的等级（L1-L5）动态调整评分权重
  - L1-L2：基础维度（破冰、识别需求）权重高
  - L3-L4：高阶维度（战略思维、关系维护）权重提升
- **动态排名**：每周更新，看你是否在进步
- **对标分析**：自动对比销冠和团队平均

**示例**：
```
破冰能力 75/100
└─ 这通电话的破冰表现：你用了"最近和 XX 公司聊过类似话题"来建立联系
   销冠通常会说："我们去年帮 3 家竞争对手解决过这个问题，
              感觉对你们也可能有帮助"
   差异分析：你的开场合理，但缺少"差异化的价值承诺"
   建议：下次加上"这对你的影响有多大"这样的追问，会更好
```

### 3️⃣ GROW 教练计划（每通电话自动生成）

基于该通电话的表现，自动生成本周的改进计划。

**G（Goal - 目标）**：
```
这周重点改进：需求识别维度（当前 62 分，销冠 88 分）
目标分数：70+ 分
```

**R（Reality - 现状）**：
```
你的平均提问数：3 个/通话
销冠平均提问数：6-7 个/通话
差距诊断：缺少深层追问，经常在第一层需求就停止了
```

**O（Options - 探索）**：
```
1️⃣ 学习销冠的提问话术
   例子："除了 A，还有其他痛点吗？"
        "这个问题通常会带来什么影响？"
        
2️⃣ 电话前准备
   准备 5 个深层追问题，列表放在手边
   
3️⃣ 电话后复盘
   每通电话后，检查：我问了几个问题？触及了核心需求吗？
```

**W（Will - 承诺）**：
```
下周验证：
✓ 每通电话至少问 5 个问题（从 3 个提升到 5 个）
✓ 尝试用"除了 A，还有..."的追问方式
✓ 到周五，需求识别维度评分达到 70+ 分
```

### 4️⃣ 客户档案自动更新

通话结束后，Clover 会自动从对话中提取客户信息，更新客户档案。

**自动更新内容**：
- 客户名称、公司、行业
- 客户的痛点和需求
- 销售阶段进展（初次接触 → 需求挖掘 → 方案演示 → 成交）
- 下一步行动和跟进时间

**用途**：
- 团队协作：其他销售能看到这个客户的完整历史
- 智能提醒：Clover 会提醒你何时应该跟进
- 数据分析：生成客户画像，找出高价值客户

### 5️⃣ 团队学习与排行（可选）

如果公司启用了团队模式，你还可以：
- 查看团队排行榜（周/月/年）
- 学习销冠的优秀通话案例
- 被 AI 推荐应该学习的人和通话

**在 Hermes 中查看**：
```
/销售 → "我们团队的周排行怎么样"
/销售 → "在破冰能力上，我应该学谁"
/销售 → "给我看一个李四的优秀通话案例"
```

## 📊 通话分析详解

### 通话分析包含的信息

#### 1. 通话转写

完整的文字版通话记录，显示销售员和客户的对话。

**使用方式**：
- 快速复习通话内容（不用再听一遍）
- 提取关键信息（客户说过什么关键词）
- 核实 AI 诊断是否准确

#### 2. 关键信息自动提取

Clover 的 NLP 模块会自动识别通话中的关键信息：

**识别项目**：

| 项目 | 作用 | 示例 |
|------|------|------|
| **痛点识别** | 找出客户的真实问题 | 客户说："我们现在的系统特别慢" |
| **需求量化** | 理解问题的严重程度 | 系统慢→导致团队效率低→成本高 50% |
| **反对意见** | 预测下步可能的反对 | 客户说："但你们价格太贵" |
| **承诺信号** | 识别客户购买意愿 | 客户说："如果能解决速度问题，我们就考虑" |
| **下一步行动** | 明确后续跟进计划 | 约好下周二 demo 时间 |

**示例**：
```
🔍 关键信息提取

痛点：系统处理速度慢（响应时间 5-8 秒）
成本：每月浪费约 200 小时（20 人 × 10 小时/人）
需求量化：时间成本 ≈ 5 万元/月
反对意见：价格（客户提到"预算紧张"）
购买信号：客户说"如果能优化到 1 秒以内，ROI 非常好"
下一步：安排技术 demo，时间待定
```

#### 3. 十维度评分详细分析

每个维度都会有**具体的分析说明**，告诉你为什么是这个分数。

**示例**：
```
🔍 需求识别维度详解（62/100）

本通电话分析：
✓ 问题识别准确：你问出了客户的主要痛点（系统慢）
✓ 问题深度初步：还能更深入
✗ 问题量化不足：没有追问成本影响、团队规模等细节
✗ 隐性需求未挖：没有问"除此之外还有其他问题吗"

对标分析：
你的提问数：3 个
销冠的提问数：6-7 个
差距：缺少 "为什么" 和 "影响" 这两类追问

改进方向：
1️⃣ 准备提问清单（情景题→困难题→暗示题）
2️⃣ 目标：把提问数提到 5-6 个
3️⃣ 下周复盘：检查提问的深度
```

#### 4. AI 问题诊断（2-3 个主要问题）

Clover 不会列出所有小问题，而是聚焦到 **2-3 个最影响成交的问题**。

**示例**：
```
🎯 本通电话的核心问题

问题 1：客户的真实采购权在谁？（未确认）
│ 现象：你和张三(采购主管)聊了，但没问决策者是谁
│ 影响：可能会谈到最后，发现决策权在总经理，前面白谈了
│ 建议：下次准备一份"影响购买决策的 5 个角色"清单

问题 2：没有量化客户的采购预算
│ 现象：客户说"预算紧张"，你接受了，没有追问具体预算
│ 影响：不知道你的价格是否真的高，还是就是想砍价
│ 建议：用话术："能否告诉我你们大概的采购预算范围？"

问题 3：成交承诺过于模糊
│ 现象：你说"我们后续会联系你"，客户说"好的"
│ 影响：没有明确下次接触的时间和方式，容易失访
│ 建议：明确说"我们下周二 10 点发送方案，
                     周三下午 2 点电话讨论，可以吗？"
```

#### 5. 改进建议（具体可执行）

不是"要多提问"这样的模糊建议，而是**具体的话术和方法**。

**示例**：
```
💡 三个改进方向（按优先级）

方向 1（本周重点）：学习 SPIN 提问法
√ 情景题：了解客户当前状况
  例：「你们现在用的是什么系统？」
√ 困难题：发现客户的问题和痛点
  例：「这个系统有什么缺点吗？」
√ 暗示题：放大问题的严重性
  例：「这个问题通常会带来什么影响？」
√ 需求题：转化为购买理由
  例：「如果能解决这个，对你们的帮助有多大？」

方向 2（下周执行）：确认购买决策链
准备清单：
□ "这个采购决策需要谁参与？"
□ "决策通常需要多长时间？"
□ "有预算限制吗？大概的范围？"
□ "如果我们能提供试用，你们什么时候能评估？"

方向 3（长期改进）：对标销冠通话
推荐学习：李四在 2026-04-20 的通话（与类似客户的通话）
他的做法：
- 开场就问了 3 个背景题
- 然后问"除了 A，还有 B、C 吗"
- 最后明确了采购时间和预算
结果：该客户 2 周内签约

你的学习方案：
□ 听一遍李四的通话，记笔记
□ 提炼出他的提问顺序
□ 下次电话前，按他的顺序准备提问
```

## 📈 十维度评分

### 十个维度的含义

| # | 维度 | 说明 | 影响阶段 | 示例 |
|---|------|------|--------|------|
| 1 | 🎯 破冰能力 | 初次接触、建立融洽 | G1 | 能否在 30 秒内消除陌生感 |
| 2 | 🔍 需求识别 | 发现和诊断客户需求 | G2-G3 | 能问出 5+ 个有深度的问题 |
| 3 | 💡 价值传达 | 说服客户方案价值 | G5 | 能否用客户语言解释你的方案 |
| 4 | 🤝 信任构建 | 建立长期信任关系 | 全程 | 客户是否愿意把你当顾问 |
| 5 | 🎭 异议处理 | 回应和化解反对意见 | G4-G6 | 遇到"太贵"时如何应对 |
| 6 | 💰 报价谈判 | 在报价上达成共识 | G6 | 能否争取合理利润 |
| 7 | 🎯 成交推动 | 促成决策、完成签约 | G7 | 能否推动客户做决定 |
| 8 | 📋 过程管理 | 管理销售周期、跟进 | 全程 | 跟进是否及时、有序 |
| 9 | 🔄 关系维护 | 售后维护、客户满意 | 签约后 | 能否保证客户满意 |
| 10 | 📊 战略思维 | 长期规划、生意管理 | 战略 | 能否识别高价值客户 |

### 分数如何计算

**三层评分体系**：

```
Layer 1：关键词识别（NLP）
  - 客户说了什么关键词（痛点、承诺、反对意见）
  - 销售员用了什么话术（提问、解释、承诺）
  - 通话感染力如何（客户是否积极响应）

Layer 2：行为模式识别
  - 提问的数量和质量
  - 说话的节奏和热情度
  - 应变和灵活性

Layer 3：对标分析
  - 与你的历史通话对比
  - 与销冠的通话对比
  - 与团队平均值对比

综合算法：
最终得分 = 关键词权重 × 行为权重 × 对标权重
         + 等级系数调整（L1-L5 的权重不同）
```

### 分数的可信度

- **总体准确度**：基于 100+ 万通销售电话训练，准确度约 92%
- **新销售员**：前 10 通电话分数波动大，从第 15 通开始稳定
- **有异议**：可以在 Hermes 中反馈，系统会记录和学习

## ❓ 常见问题

### Q1: 为什么我的通话没有立即出现分析？

**A**: Clover 需要一些处理时间：

1. **Hermes 录音上传** - 1-3 秒
2. **AI 转写** - 通常 10-20 秒（取决于通话长度）
3. **Clover 分析** - 5-10 秒
4. **生成报告** - 3-5 秒

总耗时：**20-40 秒**

如果超过 1 分钟还没出现，可能是网络问题或 API 超时，稍等一下再刷新。

### Q2: 转写有误会不会影响分析准确度？

**A**: 有一定影响，但 Clover 的分析很鲁棒：

- 如果转写准确度 > 95%，分析完全准确
- 如果转写准确度 85-95%，分析误差 < 5%
- 如果转写准确度 < 85%，Clover 会在报告中标注"转写质量需改进"

**转写不准的原因**：
- 背景音太吵（建议在安静环境通话）
- 说话太快或口音重（Hermes 需要适应）
- 专业术语太多（比如某些行业黑话）

### Q3: GROW 教练建议总是一样吗？

**A**: 不一样。Clover 会根据：
- 上一通电话的表现
- 最近 5 通电话的趋势
- 目前最低分的维度
- 整个团队的普遍问题

来生成**个性化的建议**。同一个人，每周的 GROW 建议都不同。

### Q4: 我可以和销冠比较吗？

**A**: 可以的。有两种方式：

1. **自动对比**：每通电话的分析中都会自动对比销冠
   
2. **主动查询**：在 Hermes 中问
   ```
   /销售 → "我和李四在破冰能力上有什么差异？"
   /销售 → "给我看一下李四是怎么处理价格异议的"
   ```

### Q5: 我的等级是怎么计算的？

**A**: 五级等级系统：

| 等级 | 分数 | 要求 |
|------|------|------|
| L1 | 0-40 | 通话数 ≥ 10，掌握销售基础流程 |
| L2 | 40-60 | 通话数 ≥ 20，能独立完成销售周期 |
| L3 | 60-75 | 通话数 ≥ 40，成交率 ≥ 30% |
| L4 | 75-90 | 通话数 ≥ 80，成交率 ≥ 40%，有教过他人 |
| L5 | 90-100 | 通话数 ≥ 150，成交率 ≥ 50%，行业认可 |

### Q6: 如果觉得评分不准确怎么办？

**A**: 可以在 Hermes 中反馈：

```
点击通话分析 → 右上角"有问题？" → 填写反馈

例如："这通电话的'需求识别'评分我觉得应该更高，
       因为我问了 5 个为什么，触及了客户的核心痛点"
```

系统会记录你的反馈，帮助持续优化评分模型。

### 问题 1: 通话结束后没有出现分析

**症状**：通话已结束，但 Hermes 中没有 Clover 分析结果

**解决步骤**：

1. **检查 Skill 是否安装**
   ```bash
   ls ~/.agents/skills/clover-a-sales/
   # 应该看到 SKILL.md, scripts/, package.json 等
   ```

2. **检查 .env 配置**
   ```bash
   cat ~/.agents/skills/clover-a-sales/.env
   # 确保配置格式正确，没有多余空格
   # JINA_API_KEY 是可选的，仅在需要知识库搜索时配置
   ```

3. **重建索引**
   ```bash
   cd ~/.agents/skills/clover-a-sales/scripts
   node index.js
   ```

4. **重启 Hermes 应用**

### 问题 2: 转写质量太差

**症状**：转写中有大量错别字，甚至听不出在说什么

**解决步骤**：

1. **检查通话环境**
   - 避免背景噪音（机械音、人声嘈杂）
   - 建议在安静的办公室通话
   - 避免在车上或户外通话

2. **改进说话方式**
   - 说话清晰、不要太快
   - 避免过多行业黑话
   - 定期停顿，让转写有处理时间

3. **反馈给 Hermes 团队**
   - 如果某些常用词总是转写错误
   - 可以在 Hermes 设置中教它识别这些词

### 问题 3: 十维度评分与实际不符

**症状**：觉得 AI 的评分不准确，远低于预期

**解决步骤**：

1. **查看 AI 的详细诊断**
   - 点开分析结果，查看"关键词识别"和"问题诊断"部分
   - 通常 AI 是对的，只是发现了你没注意到的问题

2. **对比销冠的类似通话**
   - 在分析中点击"参考案例"
   - 看销冠是怎么在这个阶段做的

3. **提交反馈**
   - 如果真的觉得不合理
   - 在分析页面点"反馈"，说明原因

### 问题 4: Skill 激活不了

**症状**：通话结束后，Hermes 没有调用 Clover

**解决步骤**：

1. **确认 Skill 位置**
   ```bash
   # 应该在以下位置之一
   ls ~/.agents/skills/clover-a-sales/SKILL.md
   # 或
   ls ~/.hermes/skills/clover-a-sales/SKILL.md
   ```

2. **检查 SKILL.md 格式**
   ```bash
   head -3 ~/.agents/skills/clover-a-sales/SKILL.md
   # 应该看到 YAML 前置内容
   ```

3. **重启 Hermes**
   - 完全关闭 Hermes 应用
   - 重新打开

4. **手动测试安装**
   ```bash
   cd ~/.agents/skills/clover-a-sales/
   npm run test  # 如果有测试命令
   ```
