## 🚀 快速开始

### 前置准备

✅ 已安装 OpenClaw 平台  
✅ 已通过 `bash install-openclaw.sh` 安装 Clover A-sales Skill  
✅ 配置了 `.env` 文件的 JINA_API_KEY（可选，知识库搜索需要）  

**重要**：Clover A-sales 以 OpenClaw Skill 形式运行，安装完成后**自动可用**，无需启动任何服务。

#### 📌 关于 JINA_API_KEY 配置

知识库搜索功能可选配置。如果需要让 Clover 搜索公司的销售资料、竞品信息、案例库等，需要配置 Jina AI API Key（完全免费）：

**快速获取 Jina API Key**（5 分钟）：
1. 访问 https://jina.ai
2. 邮箱或 Google 账号快速注册
3. 进入 Dashboard → API Keys → Create API Key
4. 复制生成的 key，添加到 .env 文件：`JINA_API_KEY=your_key`

**免费额度**：每月 1M tokens（足以处理 100 万+ 字的知识库）

详见 README.md 中的 [🔑 API Key 配置](../README.md#-api-key-配置) 部分。

## 🎯 核心功能与命令

### 1️⃣ 销售教练模式 `/sales` 或 `/销售`

进入 Clover 的交互式教练模式，可以进行对话、提问、获取建议。

**使用场景**：
- 🎓 学习销售方法论（Clover 八关、SPIN、Sandler 等）
- 💬 获取针对性的话术指导
- 📞 通话复盘和能力评估
- 🤝 客户阶段推进建议
- 📚 竞品应对和知识库查询

**示例对话**：
```
用户：/sales
Clover：你好！我是销售教练 Clover。今天有什么我可以帮你的？
      可选主题：
      • "我想学习破冰技巧"
      • "帮我分析一通电话"
      • "这个客户应该怎么推进"
      
用户：帮我分析一通电话
Clover：好的，请告诉我：
      1. 这通电话的客户是谁？
      2. 通话内容涉及哪些主题？（预算、需求、竞品等）
      3. 你觉得哪些方面做得好或需要改进？
```

### 2️⃣ 新手入门指南 `/新手`

为销售新人量身定制的 ToB 销售入门课程，包括基础知识、最佳实践、常见陷阱等。

**覆盖内容**：
- 🎯 Clover 八关通关框架（G0-G7）
- 💡 ToB 销售的 5 个核心方法论（SPIN、Sandler、Gap Selling 等）
- 📞 第一通电话怎么打
- 🤝 如何识别客户真正的需求
- 💰 报价和谈判技巧
- 📋 客户档案和跟进流程

**使用时机**：
- 新员工入职第一周
- 需要复习基础知识时
- 学习特定阶段的技巧时

### 3️⃣ 每日销售早报 `/早报`

每天推送 7 条精选的销售洞察、技巧、行业动态等。

**包含内容**：
- 📰 当日的销售热点（行业新闻、市场动向）
- 🎓 销售方法论小知识（1-2 分钟快速学）
- 💡 常见客户异议的应对话术
- 🏆 团队排行榜（可选）
- 📞 每日通话提示（应该关注的重点）

**使用时机**：每天早上 7-8 点，在喝咖啡时快速浏览

### 4️⃣ 销售日报引导 `/日报`

在 Clover 的引导下填写日报，自动生成总结和改进建议。

**日报内容**：
- 📱 今天进行了几通电话？与哪些客户？
- 🎯 各通电话的阶段进展（初次接触→需求挖掘→方案演示→成交）
- 💬 遇到的主要客户异议是什么？
- ✅ 你对今天的表现满意吗？为什么？
- 🔮 明天的重点是什么？

**Clover 会返回**：
- 📊 今日通话统计和质量评估
- 💡 针对遇到的异议的改进建议
- 🎯 明天的行动计划

### 5️⃣ 客户档案管理 `/客户列表`

查看所有已记录的客户档案，包括基本信息、跟进历史、阶段进展等。

**支持操作**：
- 📋 查看客户列表
- 📝 新增客户档案（自然语言：「记录一下 A 客户的信息」）
- 🔄 更新跟进记录（「A 客户今天打了电话，聊了预算的事」）
- 📊 查看跟进历史
- 🎯 查看下一步行动建议

### 6️⃣ 重建索引 `/重建索引`

当你在 OpenClaw 的知识库中添加了新文件（销售资料、竞品信息、案例库等）后，需要运行这个命令来更新 Clover 的知识库索引。

**何时需要**：
- ✏️ 上传了新的销售资料
- 📄 添加了竞品分析文档
- 📋 更新了案例库
- 🔧 知识库搜索效果不理想

## 💬 使用场景示例

### 场景 1: 破冰困难

```
用户: /sales
    我觉得自己破冰能力不太强，总是显得很生硬
    
Clover: 理解，破冰是很多销售的痛点。让我帮你诊断一下。
       
       请问：
       1. 你通常的开场白是什么？（比如："您好，我是..."）
       2. 客户的通常反应是什么？（他们听起来感兴趣吗？）

用户: 我通常就说"您好，我是来自 XX 公司的销售..."
    客户经常会打断我，说"我们现在很忙"

Clover: 明白了。这是个经典的"冷触达开场失败"。
       
       核心问题：你的开场太"自我介绍"，而不是"为客户着想"
       
       对比试试：
       ❌ "您好，我是来自 XX 公司的销售..."
       ✅ "Hi，我们最近帮 3 家同行公司解决了...的问题，
          感觉对你们也可能有帮助，能占用你 3 分钟吗？"
       
       关键差异：
       • 先说对方关心的（他们公司的问题）
       • 再说解决方案（我们帮谁解决了什么）
       • 最后说承诺（只要 3 分钟）
       
       这个方法叫"GAIN 破冰法"（Greeting → Attention → Interest → Need）
       
       你要不要先记下来，这周试试看？
```

### 场景 2: 不知道怎么推进

```
用户: /sales
    我现在跟一个客户，已经聊了两周了
    他们表示"确实感兴趣，但还要再考虑考虑"
    我不知道该怎么办了
    
Clover: 这是典型的"承诺推进"问题。
       别着急，让我帮你分析一下现在是什么阶段。
       
       根据你的描述：
       ✅ 已破冰（两周对话说明关系不错）
       ✅ 已识别需求（他们表示感兴趣）
       ❓ 现在卡在"获得推进承诺"阶段
       
       通常这个阶段的问题有两种：
       
       1. 客户不确定这个解决方案是否真的能解决他们的问题
          → 需要更多证明（案例、演示、体验）
          
       2. 客户是在礼貌地拒绝
          → 需要更深入的需求挖掘（他们真的有痛点吗？）
       
       我的建议：
       🎯 下次电话，试试这个话术：
          "我理解你想再考虑，这很正常。
           能问一下，让你有顾虑的主要原因是什么？是效果不确定，
           还是价格方面，或者说现在确实不是最好的时机？"
       
       这样问有三个好处：
       • 给出选择，让客户更好回答
       • 深入了解他们真正的顾虑
       • 为下一步改进做准备
       
       要不要看一下你们行业的成功案例，或者你和他们的通话录音？
```

### 场景 3: 想学习销冠

```
用户: /sales
    我想快速提升，能不能看看我们团队销冠是怎么做的？
    
Clover: 很好的想法！学习销冠是最快的进步方式。
       
       我可以帮你从几个角度对标：
       
       1️⃣ 他在某个维度的高分案例
          /sales → "帮我看看李四(销冠)是怎么破冰的"
          
       2️⃣ 对标他的整体能力模型
          /sales → "对比我和李四在识别需求这块的差距"
          
       3️⃣ 学习他的特定话术
          /sales → "李四处理价格异议时通常怎么说的"
       
       建议学习路径：
       📍 这周：了解销冠在你最薄弱维度的做法
       📍 下周：尝试模仿 1-2 个话术或方法
       📍 第三周：在他指导下演练，获得反馈
       
       你想看哪个维度？
```

## 📊 工作流程

### 每日工作流（销售员）

```
🌅 早上 (7:00-8:00)
  ├─ /早报              # 浏览 7 条早报内容（5 分钟）
  ├─ /销售              # 复习今天要跟进的客户话术（5 分钟）
  └─ 准备工作日
     
📞 工作时间 (8:00-17:00)
  ├─ 进行客户电话
  └─ 可以随时用 /sales 咨询遇到的问题（不用等到下班）
     
🌆 下午 (16:00-17:00)
  ├─ /日报              # 填写今日日报，获取反馈
  └─ 预览明天的重点
```

### 每周工作流（销售员）

```
📅 周一 (09:00-10:00)
  ├─ /销售              # "给我这周的 GROW 教练建议"
  │                    # 了解这周的改进重点
  └─ 制定周度目标
  
📅 周二-周四 (随时)
  ├─ /日报              # 每天填写
  └─ 根据 GROW 的 Explore 部分执行行动计划
     • 学习新话术
     • 电话前充分准备
     • 电话后及时复盘
  
📅 周五 (16:00-17:00)
  ├─ /销售              # "我这周有没有完成 Will 中的承诺？"
  └─ 自我反思和学习总结

📅 周末
  ├─ /新手              # 复习销售基础知识
  └─ /销售 → "给我推荐下周可能的客户应对方案"
```

### 每月工作流（管理者）

```
📅 月初 (1-3号)
  ├─ /销售              # "给我团队的排行榜和主要问题"
  └─ 分析团队普遍的能力缺口
  
📅 月中 (15号)
  ├─ 组织团队学习分享
  └─ /销售 → "谁在哪个维度最强，推荐给我"
  
📅 月末 (28-30号)
  ├─ /销售              # "生成月度团队报告"
```

## 📖 数据解读指南

### 等级系统（五级进阶）

| 等级 | 名称 | 总分范围 | 特征 | 目标 |
|-----|------|---------|------|------|
| L1 | 销售新人 | 0-40分 | 学习基础知识和话术 | 掌握 8 个通关步骤 |
| L2 | 初级销售 | 40-60分 | 掌握销售步骤和流程 | 能够独立完成销售周期 |
| L3 | 中级销售 | 60-75分 | 能够独立完成销售周期 | 能够指导他人，稳定成交 |
| L4 | 销冠 | 75-90分 | 能够指导他人，稳定成交 | 深化战略思维，开拓新方向 |
| L5 | 销售大师 | 90-100分 | 销售思想家，生态建设者 | 引领行业，建立方法论 |

### 十维度评分解读

**维度 1-10 的含义**：
1. 🎯 **破冰能力** - 初次接触、建立融洽的能力
2. 🔍 **需求识别** - 发现和诊断客户需求的能力
3. 💡 **价值传达** - 说服客户你的方案能解决他们问题的能力
4. 🤝 **信任构建** - 建立长期信任关系的能力
5. 🎭 **异议处理** - 回应和化解客户反对意见的能力
6. 💰 **报价谈判** - 在报价和合同上达成共识的能力
7. 🎯 **成交推动** - 促成决策、完成签约的能力
8. 📋 **过程管理** - 管理销售周期、跟进客户的能力
9. 🔄 **关系维护** - 售后维护、客户满意度的能力
10. 📊 **战略思维** - 长期规划、生意管理的能力

**分数解读**：
- **< 50分**：这是你的明显弱项，需要重点改进
- **50-75分**：常规维度，有空间改进
- **> 75分**：你的优势维度，可以教别人

## ❓ 常见问题

### Q1: 怎样快速提升某个维度的分数？

**A**: 有针对性地改进：

1. 确认这个维度的问题是什么
   ```
   /销售 → "我的破冰维度评分不高，具体问题在哪？"
   ```

2. 学习销冠在这个维度的做法
   ```
   /销售 → "我们团队谁的破冰能力最强？他们怎么做的？"
   ```

3. 获取改进建议和话术
   ```
   /销售 → "给我 3 个改进破冰的行动步骤"
   ```

4. 执行行动计划（通常 1-2 周会看到明显改进）

### Q2: 我想看数据分析和排行榜怎么办？

**A**: 使用自然语言提问：
- "我的十维度评分是多少？"
- "我们团队的周排行怎么样？"
- "我和李四(销冠)比差在哪？"
- "我这个月的通话统计"

### Q3: 什么是 GROW 教练模式？

**A**: GROW 是一个 4 步骤的教练框架：

- **G（Goal）**：这周的改进目标是什么？
- **R（Reality）**：现在的实际情况如何？
- **O（Options/Explore）**：有哪些改进选项或行动步骤？
- **W（Will）**：下周要验证什么？

Clover 会根据你的表现自动生成个性化的 GROW 建议。

### Q4: 安装后为什么找不到 Clover？

**A**: 检查以下几点：

1. **确认 Skill 已安装**
   ```bash
   # 在命令行检查
   ls ~/.agents/skills/clover-a-sales/
   # 应该看到 SKILL.md, scripts/, data/, package.json 等
   ```

2. **确认依赖已安装**
   ```bash
   # 检查 node_modules 是否存在
   ls ~/.agents/skills/clover-a-sales/node_modules/ | head -5
   ```

3. **在 OpenClaw 中尝试触发**
   - 输入 `/sales` 或 `/销售`
   - 或自然语言：「帮我分析一下这通电话」

4. **如果还是不行，检查 .env 配置**
   ```bash
   cat ~/.agents/skills/clover-a-sales/.env
   # JINA_API_KEY 是可选的，仅在需要知识库搜索时配置
   ```

### Q5: 知识库搜索返回"没有结果"怎么办？

**A**: 可能的原因和解决方案：

1. **知识库还没有相关文件**
   - 检查公司是否上传了销售资料
   - 和管理员确认知识库内容

2. **索引可能过期了**
   ```
   /重建索引    # 手动重建知识库索引
   ```

3. **尝试用更宽泛的关键词搜索**
   - 不要搜"获得承诺"，试试"承诺"
   - 不要搜"SPIN 提问"，试试"提问技巧"

### Q6: 对评分有异议怎么办？

**A**: 可以在 Clover 对话中反馈：

```
/销售 → "我觉得这通电话的'识别需求'评分不准，应该更高，因为..."
```

系统会记录你的反馈，管理员可以在后台审核和调整。

## 🔧 故障排除

### 问题 1: 输入命令后没有反应

**症状**：输入 `/sales` 或其他命令后，Clover 没有回应

**解决步骤**：

1. **确认 Skill 已正确安装**
   ```bash
   ls -la ~/.agents/skills/clover-a-sales/SKILL.md
   # 应该看到文件存在
   ```

2. **重启 OpenClaw 应用**
   - 完全关闭 OpenClaw
   - 重新打开

3. **检查 .env 配置**
   ```bash
   cat ~/.agents/skills/clover-a-sales/.env
   # 确保格式正确，没有多余空格
   ```

4. **重建索引**
   ```bash
   cd ~/.agents/skills/clover-a-sales/scripts
   node index.js
   ```

### 问题 2: 知识库搜索不工作

**症状**：问 Clover 关于公司资料的问题，返回"找不到相关信息"

**解决步骤**：

1. **确认 JINA_API_KEY 已配置**
   ```bash
   grep "^JINA_API_KEY=" ~/.agents/skills/clover-a-sales/.env
   # 应该看到 JINA_API_KEY=jina_xxx...（如果需要知识库搜索）
   # JINA_API_KEY 可选，未配置时不影响其他功能
   ```

2. **检查知识库文件是否存在**
   ```bash
   ls ~/.agents/skills/clover-a-sales/data/knowledge/
   # 应该有 *.md 文件
   ```

3. **重建知识库索引**
   ```bash
   cd ~/.agents/skills/clover-a-sales/scripts
   node index.js
   ```

4. **如果还是不行，检查索引文件**
   ```bash
   ls -lah ~/.agents/skills/clover-a-sales/data/index.json
   # 文件应该存在且大小 > 0
   ```

### 问题 3: Skill 安装后找不到

**症状**：运行了 `bash install-openclaw.sh` 但 OpenClaw 中找不到 Clover

**解决步骤**：

1. **检查安装目录是否正确**
   ```bash
   # 应该在以下位置之一
   ls ~/.agents/skills/clover-a-sales/
   # 或
   ls ~/.openclaw/skills/clover-a-sales/
   ```

2. **如果目录不存在，手动安装**
   ```bash
   git clone https://github.com/yeaphgel/clover-a-sales.git ~/.agents/skills/clover-a-sales
   cd ~/.agents/skills/clover-a-sales

   ```

### 问题 4: 性能问题（响应慢）

**症状**：Clover 的回复需要很长时间

**解决步骤**：

1. **检查知识库索引大小**
   ```bash
   du -sh ~/.agents/skills/clover-a-sales/data/
   # 如果超过 1GB，可能索引太大
   ```

2. **清理重复文件**
   ```bash
   # 找出名字里有 (1)、(2) 的文件（通常是上传重复导致的）
   find ~/.agents/skills/clover-a-sales/data/knowledge/ -name "*([0-9])*"
   # 删除重复文件，保留原始版本
   ```

3. **清理无用的知识库文件**
   ```bash
   # 删除太旧或无关的文件
   rm ~/.agents/skills/clover-a-sales/data/knowledge/old_file.md
   # 然后重建索引
   cd scripts && node index.js
   ```
