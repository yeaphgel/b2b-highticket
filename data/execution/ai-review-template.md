# Clover AI 通话复盘提示词模板

## 使用说明

1. **录音 → 转写**：用剪映/讯飞听见等工具将通话转写为文本
2. **复制下方提示词**：在 Claude/ChatGPT 中使用
3. **粘贴转写文本**：在提示词后粘贴通话记录
4. **获得分析**：AI 会输出标准化的 5 个复盘维度

## 使用技巧

### 1. 提高转写质量
- 用剪映/讯飞听见带上标点符号和断句
- 标注"销售："和"客户："，这样 AI 容易区分
- 如果转写太长（超过 5000 字），分段提交

### 2. 关键词标注法
在转写中标注关键时刻：
```
销售：你们现在怎么处理内容？
客户：我们自己写，但是……
【←← 这里开始 G2】

销售：那你们试过什么？
客户：我们试过日更，但员工说太累了，现在停更了。
【←← 痛点浮现】

销售：如果一直停更，竞品怎么办？
【←← 这是 G3 开始】
```

这样 AI 的分析会更精准。

### 3. 迭代优化
- 第 1 通电话：用基础版快速得到反馈
- 第 2-3 通电话：基于改进点测试新话术
- 第 4-5 通电话：用深度版全面优化
- 第 6+ 通电话：定期用深度版确保持续进步

## 标准输出案例

### 案例：一通中等表现的通话

```
## 📊 通关打分

| 关卡 | 得分 | 表现 | 主要卡点 |
|------|------|------|---------|
| G1 破冰 | 7 | 简单寒暄，但缺少个人化 | 没有提到客户的具体情况，显得generic |
| G2 诊断 | 6 | 问了问题，但挖掘不深 | 客户只说了"内容困难"，没有追问为什么 |
| G3 拉扯 | 4 | 没有放大问题的紧急性 | 销售直接跳到讲方案，没有"如果继续…会怎样" |
| G4 承诺 | 3 | 没有让客户承诺 | 客户说"再考虑考虑"，销售接受了 |
| G5 价值 | 5 | 讲了功能，没讲价值 | 说"我们有 AI、自动化"，没有说对客户的具体好处 |
| G6 收口 | 2 | 没有成交推进 | 结束时没有任何承诺或下一步 |
| **总分** | 4.5/10 | 诊断做了但不深，没有拉扯，成交无进展 | |

## 🎯 最大问题

- **问题**: 销售跳过了 G3 极限拉扯，直接进入讲方案
- **位置**: G2 → G3 的过渡环节
- **影响**: 客户没有感受到问题的紧急性，所以没有动力做决定
- **修正方向**: 在讲方案前，必须先用"如果…会怎样"放大问题和欲望

## 🔄 关键时刻分析

【时刻 1】
- 当时情景：销售问了"你们现在怎么做内容"，客户说"员工写但太累了"
- 销售当时说的话："我理解，所以你们需要一个自动化的方案"
- 效果：客户说"可能吧，但我们先看看"（没有紧急感）
- 应该说什么："所以，如果继续这样，员工会不会离职？离职了内容谁管？"
- 为什么更好：让客户自己想到风险，而不是销售直接给方案

## ⚡ 下次改进

- **改进点**: 在 G2→G3 的转折点，必须先做"极限拉扯"才能讲方案
- **现在的做法**: 听客户说"有困难" → 直接说"我们有解决方案"
- **改进的做法**: 听客户说"有困难" → 追问"这个困难有多严重" → "如果不解决会怎样" → "这对你们的…有什么影响" → 再讲方案
- **为什么有效**: Cialdini 的一致性原理，客户自己想到的风险，会比销售说的更有说服力
- **何时使用**: 下一通电话在 G2 结束时就开始用

## 👤 客户意向判断

- **成交可能性**: 中（50%）
- **证据**: 客户表现出了兴趣（问了几个问题），但缺乏紧急感（说要再考虑）
- **下一步行动**: D1 发一份"为什么这个问题值得现在解决"的思考框架 / D3 发一个同行业案例 / D5 发一个行业触点（"你们行业最近…"）
- **风险因素**: 如果客户一直说"再考虑"，可能永远不会决定。需要在 D7 用"如果你觉得还需要时间，我们暂停对话，一个月后再聊"来打破这个 loop
- **机会点**: 客户对"自动化"表现出了兴趣。下次强调"不需要员工，完全自动"这个价值点
```
