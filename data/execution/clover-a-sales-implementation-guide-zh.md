# Clover A-sales 销售方法论落地执行手册（中文版）

**版本**：1.0 | **适用**：OpenClaw + Hermes AI 代理 | **最后更新**：2026-05-02

---

## 快速开始（5 分钟了解核心）

### Clover A-sales 是什么？

Clover A-sales = 时间节奏（1-7-3）+ 成交八关（G1-G7）+ AI 复盘系统

**一句话**：通过 1 天锚定痛点、7 天建立信任、30 天内 3 轮促单，加上 AI 分析每次对话来加速销售个人成长。

### 为什么叫 1-7-3？

- **1**：开场 5 分钟内，找到客户 **1 个最迫切的问题**（所有其他问题都是其他问题）
- **7**：接下来 7 天，用特定的内容（诊断报告、案例、行业触点）建立信任，D7 进行"轻触收口"
- **3**：如果 7 天没成交，再花 30 天做 3 轮跟单促单（D14、D21、D30），目标 30 天内完成成交

### 八关是什么？

| 关卡 | 时长 | 目标 | 通关判定 |
|------|------|------|---------|
| **PRE** | - | 筛选对的客户 | ICP 三选二 |
| **G1** | 3 min | 建立融洽 | 客户放松，主动说话 |
| **G2** | 5 min | 诊断痛点 | 客户说出 2+ 个真实痛点 |
| **G3** | 6 min | 放大问题 | 客户情绪波动 |
| **G4** | 5 min | 获取承诺 | 客户说"现在就想解决" |
| **G5** | 4 min | 展示价值 | 客户主动问"你们怎么做到的" |
| **G6** | 4 min | 成交收口 | 获得定金或签约意向 |
| **G7** | 30 days | 跟单促单 | 启动 D1-D30 节奏 |

**关键**：跳关必败，按顺序必赢。不通关不进入下一关。

## 实施流程

### 阶段 1：准备（PRE - 赛前筛选）

#### 1.1 确认 ICP（理想客户画像）

**三选二条件**（符合两个以上就可以开谈）：
- ✓ A：已有小红书/抖音/微信账号，对增长有明确需求
- ✓ B：企业有内容运营预算（或正在考虑招聘岗位）
- ✓ C：老板本人参与会面，能直接影响采购决策

**高风险信号**（谨慎投入时间）：
- ❌ 只派员工，老板从不出现
- ❌ "让员工先试试"（说明老板没有真实决策权）
- ❌ 第一个问题就是"能不能便宜点"（说明预算不足）
- ❌ 对 AI 强烈抵触且说不出具体顾虑（说明心态有问题）

#### 1.2 Pre-Call 必做三件事

在通话前 24 小时内完成：

```
☐ 查竞品：
  看客户的小红书/抖音账号
  记下：粉丝量、发布频率、内容质量、最近停更多久
  
☐ 找焦虑：
  搜索对方行业最近的竞品动向
  找到 1 个"如果他们不改进，会被甩开"的触发点
  
☐ 准备案例：
  找到 1 个同行业的成功案例（最好粉丝量、客户体量相近）
  准备 3-5 个关键数据（前vs后）
  放进"口袋备用"
```

**在 OpenClaw 中的操作**：
- 在客户的 CRM 档案中记录 Pre-Call 三件事
- 标记客户的 ICP 评分（3 选几个）
- 标记高风险信号（如果有）

### 阶段 2：开场到成交（G1-G6）

#### 2.1 G1 破冰暖场（3 分钟）

**目标**：对方放松，主动多说话，感觉是和朋友聊天

**话术框架**：
```
"XX 总，感谢今天抽出时间。我提前看了一下你们的账号，
[具体数据：粉丝量多少、发布频率如何、内容质量怎么样]，
你们在内容上其实是有基础的，就是缺一个稳定的机制。

今天我想先了解一下你们目前的情况，
看看我们能不能帮上忙——帮不上的话我也会直说。"

→ 目的：展示"有备而来"（权威性）+ 降低防御心（顾问感）
```

**禁止动作**：
- ❌ 开口就介绍产品或系统架构
- ❌ 说"我们的 AI 非常先进"
- ❌ 上来就问"你们有预算吗"

**通关判定**：客户放松，开始主动说话

**在 Hermes 中的优化**：
- 事前：Hermes 可以扫描客户的社媒账号，生成"看起来很专业的 Pre-Call 观察"
- 关键话术：让 Hermes 建议"基于这家企业的数据，最有说服力的开场是什么"

---

#### 2.2 G2 诊断现状（5 分钟）

**目标**：客户用自己的语言说出 2 个以上的真实痛点

**黄金问题库**（按顺序问，但根据客户答案灵活调整）：

```
Q1：你们现在内容这一块，主要是谁在负责？
→ 了解现状人力结构

Q2：0 到 10 分，你对现在账号的增长速度满意吗？
→ 了解满意度（如果是 8-10 分就可能不是理想客户）

Q3：你们试过自己发内容吗？为什么停下来了？
→ 挖掘真实痛点（"停下来"是最大的信号）

Q4：竞品里有没有哪个账号让你觉得"他们怎么做到的"？
→ 制造焦虑（对标）

Q5：如果内容这块一直这样下去，对你们今年的获客目标有什么影响？
→ 量化问题的影响
```

**关键技巧**：问完问题，就闭嘴。让客户说话。

**通关判定**：客户说出 2 个以上真实痛点（用客户自己的语言）

**在 Hermes 中的优化**：
- 实时转录：Hermes 听转写，如果发现客户说出了痛点，标记
- 追问建议：如果客户答案不够深，Hermes 建议下一个追问
- 诊断报告：G2 结束后，Hermes 可以自动整理"诊断发现"

---

#### 2.3 G3 极限拉扯（6 分钟）

**目标**：放大焦虑 × 放大欲望，制造认知差距

**成交公式**：成交 = 欲望 × 信任 ÷ 阻力

**向下拉扯**（放大痛苦）：
```
"你说员工负责内容，但他还要做什么？
 如果他明天离职，这一块是不是直接断了？
 你们有系统去承接这个风险吗？"

"竞品那个账号去年才开始做，现在已经 15 万粉。
 这个差距明年会更大吗？
 等他们到 50 万粉时，你们还有翻盘的机会吗？"

"如果内容一直没有稳定产出，今年的获客目标怎么办？
 这对你们的年度业绩有多大影响？"
```

**向上拉扯**（放大欲望）：
```
"如果这块问题解决了，你们的获客渠道会发生什么变化？

想象 6 个月后，账号每周稳定涨 2000 粉，
进来的都是精准客户——对业务影响多大？

你自己完全不用管内容了，
那个时间你可以花在哪里？战略？新市场？"
```

**通关判定**：客户出现明显情绪波动（要么担忧，要么兴奋）

**在 Hermes 中的优化**：
- 转写分析：检测客户的语气变化（速度变快、重复某个词等）
- 情绪打分：G3 是否成功可以看"客户是否表现出了紧迫感"
- 下一步建议：如果客户情绪反应不明显，建议销售哪种拉扯更有效

---

#### 2.4 G4 引导承诺（5 分钟）

**目标**：让客户自己说出"我想改变"的承诺

**引导脚本**（四步走）：

```
Step 1 - 总结归因：
"所以你的情况是：想做内容但缺稳定机制，
 导致账号停滞而竞品在加速——我理解对吗？"
 
→ 目的：让客户"承诺"已经讲清楚的现状

Step 2 - 扩大差距：
"如果继续用现在的方式，这个问题 6 个月后能自己解决吗？"

→ 目的：让客户承诺"不改变就没法解决"

Step 3 - 获取承诺（最关键）：
"那你是想找到方法彻底解决，还是继续先这样？"

→ 这一句最关键。客户的答案决定成交概率：
  如果说"想解决" → 成交概率 75%+
  如果说"再考虑" → 成交概率 25%以下

Step 4 - 确认时机：
"如果有方案能解决这个问题，你希望什么时候开始？"

→ 了解客户的 timeline，是真需求还是"礼貌性听听"
```

**通关判定**：客户明确说出"现在就想解决" 或 "下周开始" 等具体时间

**在 Hermes 中的优化**：
- 关键字识别：Hermes 自动检测客户是否说出了承诺信号
- 风险识别：如果客户说"再考虑考虑"，标记为"低概率客户"
- 建议下一步：如果 G4 失败，建议是做 G7 跟单还是放弃这个线索

---

#### 2.5 G5 价值展示（4 分钟）

**目标**：展示最短路径，引发主动追问

**核心原则**：只讲结果，不讲过程。用案例，不用功能。

**三板斧框架**：

```
① 定位专家身份：
"我们只服务[垂直行业]，做精不做广。
 所以我们的方法对你们是匹配的。"

② 案例结果证明：
"跟你们情况类似的[行业]企业，3 个月账号从 0 到 1.2 万粉，
 带来了 23 个到店询价。"

③ 交付方式差异：
"你不用管内容生产，每周五收到下周所有内容，
 花 10 分钟审批——仅此而已。
 我们负责创意、排期、发布、数据追踪。"
```

**关键**：不要说"我们有 AI、自动化…" → 客户不在乎
**要说**："你不用操心，每周 10 分钟，其他都是我们的"

**通关判定**：客户主动追问"你们怎么做到的" 或 "这个流程怎么操作"

**在 Hermes 中的优化**：
- 案例库查询：Hermes 从案例库找到"最匹配的同行企业案例"
- 行业对标：Hermes 生成"基于这个企业的数据，最相关的案例"
- 结果量化：Hermes 建议"这个案例中最打动客户的 3 个数据是什么"

---

#### 2.6 G6 成交收口（4 分钟）

**目标**：报价、处理异议、拿定金

**报价话术**：
```
"好的，我跟你说一下我们这边的方案。

[方案名称]，每个月 X 元，包含[核心交付内容]。

说完价格后——停下来，什么都不要说。
等对方开口。"

← 沉默是金。很多销售在这里说太多，反而破坏成交。
```

**异议处理手册**（用 Sandler 反向法）：

| 对方说…… | 反向的问法 |
|---------|----------|
| **价格太贵了** | "贵是相对什么说的——相对一个全职员工，还是相对什么都不做？" |
| **让员工先试试** | "完全支持。我建议设个 30 天节点。30 天后我们一起看三个数据：发了多少条、粉丝涨了多少、员工每周花了多少时间。你来判断值不值得。" |
| **再考虑考虑** | "没问题。那你主要在考虑哪个方面——是价格、效果、还是内部需要讨论？" |
| **效果能保证吗** | "我不承诺数字，我承诺执行：每周内容不断档、每月一份体检报告、连续 3 周数据下滑我们主动出策略调整。这样可以吗？" |

**收口推进句**（小步成交）：
```
"好，那就这样——你这边出个意向金，
 我们启动诊断期，先把你们的资料整理一下。
 不合适，原路退回。你看可以吗？"

或

"你刚才说想现在解决——那我们今天就把第一步走起来，如何？
 签这份试用协议，付这个定金，我们下周启动诊断。"
```

**通关判定**：收到定金或签约意向

**在 OpenClaw 中的操作**：
- 在客户档案中记录"成交金额、签约日期、启动时间"
- 标记"已进入 G7 跟单系统"
- 设置 D1 提醒

### 阶段 3：跟单促单（G7 - 1-7-3 节奏）

#### 3.1 D1-D7：第一周（信任闭环）

| 时间 | 动作 | 具体内容 | 目的 |
|------|------|---------|------|
| **D1** | 感谢 + 诊断价值 | 发送"竞品分析报告" + 感谢信 | 不催单，送价值；建立互惠感 |
| **D3** | 行业触点 | "你们行业昨天有条笔记涨了 3000 粉，我看了一下结构，和你们定位很匹配…" | 展示你在持续关注他们的行业；社会证明 |
| **D5** | 案例证明 | "我们一个客户的内容本周审批通过了，数据出来分享给你…" | 社会证明；制造"别人在用，你不用会落后"的感受 |
| **D7** | 轻触收口 | "差不多一周了，你那边有什么新想法吗？我这边有个名额可以给你留着" | 检查进度；创造稀缺性（名额限制） |

**D1 内容模板**：
```
【邮件主题】：XXX 企业 × 内容运营诊断报告

【邮件正文】：
感谢 XX 总今天的时间。

根据我们的分析，贵公司在内容运营上的现状是：
✓ 账号基础：[粉丝量、发布频率等]
✗ 主要瓶颈：[客户说出的痛点]
△ 对标差距：[竞品 vs 你们]

随附：竞品分析详报（PDF）

我的建议是：[一句话建议]

有任何问题，随时可以联系我。
```

**D3 内容示例**：
```
顺便说一句，我昨天看到你们行业有个案例，
《XXX 品牌如何用内容驱动 3 个月增长 10 倍》

我简单看了一下他们的内容结构和选题策略，
和你们的品牌定位其实很匹配。

要不要我把这个案例的拆解发给你？
有几个细节可能对你们的运营有启发。
```

**D5 内容示例**：
```
好消息！

我们最近的一个客户（也是做[产品/服务]的）
他们的内容本周通过了审批，已经开始发布。

初步数据很不错：
- 发布后 3 天获赞 800+
- 评论 120+
- 转化到门店询价 12 个

他们用的是同样的方法论。
有兴趣看一下完整数据吗？
```

**D7 内容示例**：
```
差不多一周了，想问一下：

上次的诊断报告有帮助吗？
你和团队讨论过接下来的方向了吗？

对了，我这边这个月还有一个名额，
可以给你留到下周三。
之后的名额会到下个月，可能会需要等一段时间。

你那边什么时候有结论？
```

#### 3.2 D14-D30：第二、三周（持续价值 + 促单）

| 时间 | 动作 | 具体内容 |
|------|------|---------|
| **D14** | 第二轮价值输出 | "刚好有个行业节日，我帮你做了一个内容策划草稿，你看符不符合思路…" |
| **D21** | 第三轮收口 | "XX 总，你上次说等一段时间。现在有结论了吗？方案还给你留着。" |
| **D30** | 最终判断 | "若 30 天三轮均无回应，归档为长期培育，每月一次行业报告维持存在感" |

**在 OpenClaw 中的自动化**：
- 设置自动提醒：D1、D3、D5、D7、D14、D21、D30
- 内容库调用：OpenClaw 的内容库自动生成"行业触点"和"案例证明"
- 邮件自动发送（但建议人工审核一下，加上个人化语言）
- 进度追踪：客户的打开率、回复情况等

## 在 OpenClaw 中的完整工作流

### CRM 字段设置

在每个客户档案中记录：

```
【基础信息】
- 客户名称、公司、行业
- 联系人（老板、运营负责人等）
- 联系电话、邮箱

【ICP & 预选】
- ICP 评分（1-3，符合几个条件）
- 高风险信号（有无）
- Pre-Call 三件事完成情况

【销售进度】
- 当前关卡（PRE / G1 / G2 / ... / G7）
- 通关打分（0-10）
- 最大问题（本次对话的卡点）
- 预期下次对话时间

【跟单日程】
- G1 对话时间、关键内容
- G2 诊断结果、客户痛点
- G3-G6 各环节进展
- G7 跟单日程（D1、D3、D5 等）

【成交信息】
- 成交金额、付款进度
- 启动时间、合同日期
```

### Hermes AI 代理的角色

**Hermes 可以做什么**：

| 任务 | Hermes 的优化 |
|------|--------------|
| **通话前准备** | 自动扫描客户社媒、生成 Pre-Call 观察 |
| **通话中** | 实时转写 + 关键词检测 + 建议追问 |
| **通话后** | 自动生成"诊断报告"、标记各个 G 的完成度 |
| **跟单准备** | 根据客户行业，推荐最匹配的案例和行业触点 |
| **邮件生成** | 基于对话内容，生成个性化的 D1、D3、D5 邮件 |
| **进度预警** | 如果发现客户成交概率低，预警销售 |

**Hermes 的提示词示例**：
```
你是一名销售教练。根据这段通话转写，用 Clover A-sales 框架分析：
1. 当前在哪个关卡（G1-G6）
2. 每个关卡的得分（0-10）
3. 最大的卡点是什么
4. 下一步应该怎么做

【通话转写】
[转写内容]
```

## 实战检查清单

### 每通电话前

- [ ] Pre-Call 三件事完成（查竞品、找焦虑、准备案例）
- [ ] CRM 中输入了客户基础信息
- [ ] 确认了客户的 ICP 评分
- [ ] 准备了开场的"具体数据"（不是generic的问候）

### 每通电话后

- [ ] 记录了当前的关卡（G1-G6）
- [ ] 给每个关卡打了分（0-10）
- [ ] 标记了"最大问题"（卡点在哪）
- [ ] 用 AI 复盘模板分析了一遍通话
- [ ] 记录了"下次改进"（具体的话术改变）
- [ ] 设置了下次跟进的时间和方式

### 成交后

- [ ] 客户状态更新为"已成交"
- [ ] 记录了成交金额、合同日期
- [ ] 启动了 G7 跟单系统（D1 提醒已设置）
- [ ] 在 Hermes 中标记了"成交信息"，用于后续复盘

### 周度回顾（每周一次）

- [ ] 本周对话数、成交数
- [ ] 平均通关得分（对标上周是否在进步）
- [ ] 最常见的卡点是什么（可能需要整体优化）
- [ ] 有没有"高概率客户"被流失（为什么）
- [ ] 有没有"低概率客户"意外成交（原因是什么）

## 常见问题解决

### Q1：客户说"我们需要先试试看"，怎么办？

**标准回答**（Sandler 反向法）：

```
销售："完全支持。不过试用的话，我建议我们设个 30 天的节点。

30 天后，我们一起看三个数据：
1. 发了多少条内容（质量怎么样）
2. 粉丝增长了多少
3. 你们员工花了多少时间管理

根据这三个数据，你来判断是否继续。公平吗？"

客户："好的。"

销售："那我们今天就把试用协议签了，付个意向金，下周启动。可以吗？"
```

**心理学原理**：把"随意试用"变成"有界限的试用"，
一旦设定了具体的判断标准，客户会更认真对待。

### Q2：客户问"你们的价格是多少"，应该什么时候说？

**标准做法**：

```
不要在 G1-G2 就说价格。

最晚在 G5 之前说。
最好在 G5 之后（客户已经看到价值了）。

说价格的时机：
❌ 错：G1 就说
❌ 错：G2 问价格就直接说
✓ 对：G5 客户已经对价值有认知后再说

说价格的方式：
销售："我们的服务通常是 X 元/月。
        说完后停顿，什么都不说。"
        
        （让客户先开口）
```

### Q3：D7 没有得到回应，怎么办？

**标准处理**：

```
D7 无回应时（通常是说"再考虑"或没有回复）：

选项 A（如果客户有回应但说"再考虑"）：
"没问题。那一个月后我们再聊？到时候我有新的案例给你看。"
→ 设置 D30 的轻接触跟进

选项 B（如果客户完全没有回应）：
"我想给你发一份新的行业报告。不是推销，就是分享一些趋势。"
→ 用"免费价值"维持存在感

D14-D21 再尝试 1-2 次。
如果 D30 还是没有信号，归档为"长期培育"。
每个月送一次"行业报告"，维持关系温度。
```

### Q4：如何判断一个客户是"真需求"还是"假咨询"？

**判断标准**：

```
真需求的信号：
✓ G4 说出了"现在想解决"
✓ 问具体的实施细节（不是笼统的产品介绍）
✓ 询问时间表（"下周能启动吗"）
✓ 问团队配合的细节（"我们需要配合什么"）

假咨询的信号：
✗ G2-G3 都过不了（不承诺问题的存在或紧迫性）
✗ 一直说"再考虑"（G4 没有明确承诺）
✗ 问的都是"你们有什么功能"（没有自己的需求）
✗ 多次约不上老板（老板不重视）

处理方式：
- 真需求 → 投入，用 Clover A-sales 完整流程
- 假咨询 → 早期识别后，进入"长期培育"模式（低频跟进）
```

## 成功案例记录

### 案例 1：3 通电话成交（高概率客户）

```
D1：通话
- G1：建立融洽 ✓
- G2：客户说出 3 个痛点 ✓
- G3：客户明显表现出焦虑 ✓
- G4：客户说"现在想解决" ✓
- G5：客户问"怎么实施" ✓
- G6：成交，付定金 ✓

成交周期：通话当天

原因分析：
1. Pre-Call 做得好，开场就有权威感
2. 客户确实有真实需求（从 G2 就很明显）
3. G3 拉扯非常有效，客户情绪波动明显
4. 成交没有异议（客户很信服）

从这个案例学到的：
- 权威感的建立非常关键（决定了客户的防御心）
- 真需求客户的信号非常明显（在 G2 就能看出）
```

### 案例 2：30 天周期成交（中等客户）

```
D1：第一通电话
- G1-G6 完整走完，但 G4 时客户说"再想想"
- 成交概率：中等（50%）

D7：D1+6 天
- 发送了诊断报告 + 行业触点
- 客户回复说"有帮助，但还在考虑"

D14：跟进
- 发送了同行案例
- 客户主动问起实施流程

D21：关键跟进
- 客户说"老板最近在考虑，可能下月启动"
- 销售强调了"现在锁定价格比较划算"

D30：最终促单
- 客户终于确认"下周启动"
- 签约，付定金

从这个案例学到的：
- 30 天跟单系统确实有效
- D14-D21 的"持续价值"（案例+行业报告）很关键
- 稀缺性（价格/名额）在D21 时最有效
```
