# SPIN Selling - 深度诊断销售法

**作者**：尼尔·雷克汉姆（Neil Rackham） | **出版**：1988 | **应用**：高价值 B2B 成交（$50K+）

---

## 核心框架：SPIN 四问法

### 为什么需要 SPIN？

普通销售的问题：说得太多，听得太少 → 客户防御加强

SPIN 的优势：系统化提问 → 激发客户自我认知 → 自我说服

### SPIN 四步骤

#### 1️⃣ S - Situation 情景题（建立共识）

**目的**：了解客户现状，建立基础信息

**特征**：事实性问题，客户容易回答

**举例**：
```
"你们现在内容这块，主要由谁负责？"
"一周发几次内容？"
"目前用什么工具来管理?"
```

**Clover 中的对应**：G2 诊断现状的前两个问题

---

#### 2️⃣ P - Problem 困难题（挖掘真实痛点）

**目的**：找出客户遇到的问题和障碍

**特征**：开放式问题，需要客户深度思考

**举例**：
```
"你们现在发内容遇到什么困难吗？"
"这些困难对业务有什么影响？"
"为什么还没有解决这个问题？"
```

**关键**：不是"有没有问题"，而是"什么问题让你最困扰"

**Clover 中的对应**：G2 诊断现状的黄金问题库

---

#### 3️⃣ I - Implication 暗示题（让客户自己说出严重性）

**目的**：引导客户认识到问题的后果

**特征**：连锁提问，一个问题引出下一个问题

**举例**：
```
P: "内容停更一个月会怎样？"
I: "如果停更意味着粉丝不增长，这会影响你们今年的增长目标吗？"
I: "增长目标没达成，对公司的业绩有什么影响？"
I: "那你觉得这个问题有多紧急？"
```

**为什么有效**：
- 客户自己说出问题的严重性，比销售说"我们能解决"有 10 倍的说服力
- 创造了"欲望"（成交心理学的分子）

**Clover 中的对应**：G3 极限拉扯中的"向下拉扯"

---

#### 4️⃣ N - Need-Payoff 需求回顾题（聚焦解决方案价值）

**目的**：让客户自己说出解决方案的价值

**特征**：正向提问，引导客户说出期望结果

**举例**：
```
"如果能稳定发内容，你觉得对账号增长有什么帮助？"
"有了稳定的内容产出，你自己能腾出多少时间来做战略？"
"如果内容做好了，能带来多少精准客户询价？"
```

**为什么有效**：
- 客户自己说出了方案的好处
- 比销售说"我们的产品有什么功能"强 100 倍
- 建立了"承诺"（客户已经承认解决方案的价值）

**Clover 中的对应**：G4 引导承诺 + G5 价值展示

## SPIN 的销售大忌

### ❌ 不要：问太多情景题

**错误的对话**：
```
销售：你们现在几个人负责内容？
客户：3 个人
销售：你们一周发几次？
客户：2 次
销售：用什么工具？
客户：飞书和石墨
```
→ 客户觉得这是"问卷调查"，不是"专业顾问诊断"

**正确做法**：
- 情景题只问 2-3 个关键问题
- 快速进入困难题（P）
- 用暗示题放大问题（I）
- 用需求回顾题确认方案价值（N）

---

### ❌ 不要：告诉客户问题的严重性

**错误**：
```
销售："你不发内容太危险了，竞品都在发，你会被甩开。"
```

**正确**：
```
销售："如果竞品每周发内容而你停更，这个差距会怎样？"
客户："肯定会被甩开。"
销售："那这个问题对你来说有多紧急？"
```

**为什么**：人无法反驳自己。让客户自己说出问题的严重性，远比你说有说服力。

---

### ❌ 不要：跳过困难题直接问需求

**错误的顺序**：
```
销售：S1 → S2 → N1 → N2
结果：客户觉得"这个销售在推销，没有诊断"
```

**正确的顺序**：
```
销售：S1 → S2 → P1 → P2 → I1 → I2 → N1 → N2
结果：客户觉得"这个人真懂我们的问题，是顾问"
```

## SPIN 在 Clover 中的应用

### Clover 是 SPIN 的升级版本

| SPIN (1988) | Clover (2026) |
| --- | --- |
| 理论型问诊法 | +时间节奏（1-7-3） |

- **S 情景题** = Clover G2 的前两个问题
- **P 困难题** = Clover G2 的黄金问题库

## SPIN 话术实战库

### 情景题（Situation）

```
"你们现在怎么处理内容这一块？"
"这块主要是谁在负责？"
"一周大概能发多少条内容？"
"从零到一个月涨多少粉？"
```

### 困难题（Problem）

```
"这样的频率会不会遇到什么困难？"
"为什么停更了？"
"坚持发内容最大的挑战是什么？"
"现在账号增速如何？满意吗？"
```

### 暗示题（Implication）— 关键技巧

**链式追问**：一个问题引出下一个

```
P: "员工写内容没时间？"
I: "那是不是意味着内容优先级不高？"
I: "优先级不高，是不是因为现在效果不明显？"
I: "效果不明显，对今年目标影响多大？"
```

**从痛苦延伸到后果**：

```
P: "竞品在加速，你们在停滞？"
I: "这个差距明年会更大吗？"
I: "更大的差距意味着什么——获客更难？"
I: "获客更难对公司意味着？"
```

### 需求回顾题（Need-Payoff）

```
"如果稳定发内容，对账号有帮助吗？"
"有了稳定产出，你能省出多少时间？"
"内容做好了，能带来多少询价？"
"你觉得解决这个问题，对公司最大的收益是什么？"
```

## SPIN 的终极心法

> **"Stop selling. Start diagnosing."**

当你在问诊而不是推销时，客户的防御心就消失了。当客户自己说出了问题的严重性和解决方案的价值时，成交就不是推销，而是顺势而为。
