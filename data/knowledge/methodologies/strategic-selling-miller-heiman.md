# Strategic Selling - 大订单多决策人管理

**创始人**：罗伯特·米勒 & 史蒂芬·海曼（Miller Heiman） | **适用**：高价值 B2B 大订单（¥500K+） | **核心问题**：大订单有 5-8 个决策人，不是一个人说了算

---

## 核心洞察：你面对的不是一个人，而是一个决策网络

### 为什么大订单总是"卡住"？

**普通销售的假设**：
```
假设：搞定老板 = 搞定订单
现实：70% 的大订单丢单不是因为"老板说不"，而是因为"其他角色暗中反对"
```

**Strategic Selling 的假设**：
```
每个大订单背后都有 4 种关键角色
→ 你需要对每种角色有不同的策略
→ 你需要找到"内部拥护者"帮你推动决策
→ 你需要识别"内部反对者"并化解阻力
```

### 大订单的决策复杂度

| 订单规模 | 决策人数 | 平均决策周期 | 丢单主因 |
|---------|---------|------------|---------|
| ¥50K 以下 | 1-2 人 | 1-4 周 | 价格 |
| ¥50K-¥500K | 2-4 人 | 4-12 周 | 需求不匹配 |
| ¥500K 以上 | 4-8 人 | 12-24 周 | 内部反对者 |

**结论**：订单越大，越需要多角色策略，而不是更强的销售技巧

## 四种关键角色（Buyer Types）

### 角色 1：Coach（教练/内部指导者）

**定义**：愿意帮助你的人，提供信息和内部情报

**特征**：
- 主动给你内部信息
- 告诉你决策流程是怎样的
- 提醒你潜在风险
- 希望你成功

**如何识别 Coach**：
```
"我可以告诉你，老板最近最关心的是..."
"你们这个方案，技术那边可能会有顾虑..."
"我建议你在下次演示时，重点讲这一点..."
```

**Clover 中的对应**：= G1 破冰后开始建立信任的关键人物，但 Strategic Selling 强调要尽早确认谁是你的 Coach

---

### 角色 2：Champion（内部拥护者）

**定义**：在内部会议上为你说话的人，真正的支持者

**特征**：
- 愿意在内部会议上推荐你
- 能影响其他决策人
- 有自己的利益诉求（你的方案帮他解决问题）
- 在决策层有话语权

**Coach vs Champion 的区别**：
```
Coach：给你信息 → 被动帮助
Champion：帮你推动 → 主动推动

Coach 可能只是"友善"，Champion 才是"站你这边"
→ 很多销售混淆了这两个角色，导致丢单
```

**如何培养 Champion**：
```
1. 找到谁最受益于你的方案（个人利益绑定）
2. 给他足够的弹药（数据、案例、对比）让他在内部会议上使用
3. 帮他预演反对意见，准备好回应
4. 让他觉得"这个项目成功 = 他的成功"

实战话术：
"张总，我理解你在这个项目上担了很大风险。
我想帮你准备一下下周的内部评审。
这是 3 个你可能被问到的问题和回应，你看看有没有用？"
```

**Clover 中的对应**：= G5 价值展示中的"案例证明"，但 Strategic Selling 强调要让客户变成你的"内部代言人"

---

### 角色 3：Competitor（内部反对者/竞争对手）

**定义**：在内部反对你的方案的人

**注意**：Competitor 不是外部竞品销售，而是客户公司内部的反对声音

**特征**：
- 提出反对意见
- 支持其他方案（包括"什么都不做"）
- 可能来自利益受损的部门
- 可能来自不信任新方案的技术人员

**如何识别 Competitor**：
```
"我们以前用过类似的东西，效果不好。"
"这个方案听起来很好，但技术上可能实现不了。"
"我们现在这样也挺好，没必要改变。"
```

**应对策略**：
```
1. 不要忽视：Competitor 的意见会直接影响决策
2. 不要对抗：直接对抗会让对方更坚定
3. 通过 Coach 了解：Competitor 的真正顾虑是什么
4. 对症下药：技术顾虑 → 技术演示；利益冲突 → 双赢方案

实战话术（通过 Coach 了解）：
"张总，我听说李经理对这个方案有些顾虑。
能具体说说他担心什么吗？我想直接跟他聊聊，看看能不能解决他的问题。"
```

**Clover 中的对应**：= G6 异议处理的扩展版，但 Strategic Selling 强调要在异议出现前就识别和化解

---

### 角色 4：Economic Buyer（经济决策者）

**定义**：最终拍板的人，控制预算

**特征**：
- 有最终决定权
- 关心 ROI 和投资回报
- 通常不关注技术细节
- 决策依据：战略契合度 + 财务回报

**如何识别 Economic Buyer**：
```
问 Coach："这个项目最终由谁拍板？"
问 Champion："如果我们都同意，谁能最终签字？"

注意：Economic Buyer 可能不是跟你直接对话的人
→ 很多销售跟了 3 个月才发现找错人了
```

**如何打动 Economic Buyer**：
```
不要讲：产品功能、技术架构、操作流程
要讲：ROI 数据、战略价值、风险控制

实战话术：
"王总，这个方案预计每年能帮公司节省 ¥200 万，
同时提升 15% 的客户留存率。
投资回报率预计是 3:1，6 个月内回本。"
```

**Clover 中的对应**：= G4 承诺中的"确认时机"，但 Strategic Selling 强调要提前识别真正的经济决策者

## 决策流程地图（Winning Plans）

### 为什么要画决策流程地图？

**问题**：70% 的销售丢单是因为"不知道决策流程是什么"

**Strategic Selling 的解决**：
```
在每个大订单开始前，画出完整的决策流程图：
谁参与 → 谁影响 → 谁拍板 → 什么流程 → 什么时间

这张图就是你的"作战地图"
```

### 决策流程地图模板

| 角色 | 姓名 | 职位 | 权力类型 | 对你的态度 | 关键关注点 | 应对策略 |
|------|------|------|---------|-----------|-----------|---------|
| **Coach** | 张明 | 运营经理 | 信息权 | 支持 | 个人绩效 | 提供弹药，帮他成功 |
| **Champion** | 李强 | 副总 | 影响权 | 强力支持 | 部门 KPI | 培养内部代言人 |
| **Economic Buyer** | 王总 | CEO | 拍板权 | 中立 | ROI+战略 | 直接讲数据和战略价值 |
| **Competitor** | 赵工 | CTO | 否决权 | 反对 | 技术风险 | 通过 Coach 化解顾虑 |
| **User** | 小陈 | 主管 | 使用权 | 中立 | 易用性 | 演示 + 试用 |

### 如何填充决策流程地图？

**Step 1：向 Coach 提问**
```
"这个项目还有谁会参与决策？"
"你们公司一般怎么做这类决定？"
"最后签字的是谁？"
"之前有没有做过类似的采购？流程是怎样的？"
```

**Step 2：识别每个人对你的态度**
```
支持（+）：主动帮助你，希望方案成功
中立（0）：不反对也不支持，取决于证据
反对（-）：有顾虑，可能阻碍决策
未知（?）：你还没接触过
```

**Step 3：制定应对策略**
```
对支持者：强化信心，提供弹药
对中立者：提供证据，消除顾虑
对反对者：通过 Coach 了解真正原因，对症下药
对未知者：尽快建立联系
```

## Strategic Selling 的 6 步流程

### 第 1 步：定义你的 Winning Result

**目标**：明确什么对你来说算"赢"

```
不是"签单"，而是具体的、可衡量的结果：
"3 个月内，以 ¥500K 的价格签下 ABC 公司的年度合同"

这个定义必须具体到：
- 金额
- 时间
- 合同范围
```

---

### 第 2 步：画出决策流程地图

见上方"决策流程地图"章节。

---

### 第 3 步：识别关键角色的策略

**对每个角色问 4 个问题**：
```
1. 这个人对我是什么态度？（+ / 0 / - / ?）
2. 我需要他对我是什么态度？
3. 我怎样才能影响他？
4. 我需要 Coach 帮我做什么？
```

**关键洞察**：
```
Economic Buyer 不需要被"说服"，他需要被"证明"
→ 给他看 ROI、案例、数据

Champion 需要被"武装"
→ 给他数据、案例、对比，让他在内部会议上能用

Competitor 需要被"理解"
→ 通过 Coach 了解他的真正顾虑，然后对症下药
```

---

### 第 4 步：培养 Champion

**Champion 培养清单**：
```
- [ ] 确认这个人有内部影响力
- [ ] 确认他的利益与你的方案一致
- [ ] 给他 3 个以上可以在内部会议上使用的数据/案例
- [ ] 帮他预演可能被问到的问题
- [ ] 确认他愿意在内部会议上推荐你
- [ ] 确认他能接近 Economic Buyer
```

---

### 第 5 步：接触 Economic Buyer

**为什么不能跳过 Champion 直接找 Economic Buyer？**
```
直接去找 Economic Buyer：
→ 没有内部人推荐，对方会防御
→ 你不够了解决策流程
→ 你不知道 Economic Buyer 真正关心什么

正确路径：
Coach 提供情报 → Champion 帮你安排 → 你带着证据去见 Economic Buyer
```

**Economic Buyer 会面准备清单**：
```
- [ ] 了解他的个人背景（行业、经历、关注点）
- [ ] 准备 3 个核心 ROI 数据
- [ ] 准备 1 个同行业成功案例
- [ ] 准备 1 页纸的 executive summary
- [ ] 通过 Champion 确认他会问什么问题
- [ ] 确认会面时间不超过 30 分钟
```

---

### 第 6 步：管理决策流程

**目标**：控制节奏，不被"考虑考虑"卡住

```
错误做法：
"好的，您考虑一下，我下周再联系您。"

正确做法：
"王总，理解你需要时间评估。
这样，我们定个时间——下周三下午 3 点，
我带技术团队来做一个 1 小时的深度演示。
在那之前，我会把今天的讨论总结发给你。
你看可以吗？"
```

**时间节奏管理**：
```
Week 1-2: 建立 Coach 关系，了解决策流程
Week 3-4: 培养 Champion，准备内部演示
Week 5-6: 见 Economic Buyer，提交方案
Week 7-8: 处理异议，跟进所有角色
Week 9-10: 成交收口
Week 11-12: 签约
```

## Strategic Selling vs SPIN vs Sandler vs Clover

| 维度 | SPIN | Sandler | Strategic Selling | Clover |
|------|------|---------|-------------------|--------|
| **核心问题** | "你有什么问题？" | "你值不值得我帮？" | "谁说了算？谁支持你？谁反对你？" | "1-7-3 节奏推进" |
| **适用场景** | 单人决策，$50K+ | 单人决策，需要筛选 | 多人决策，¥500K+ | 全场景 |
| **关键角色** | 不区分 | 不区分 | Coach/Champion/Competitor/EB | 不区分 |
| **决策周期** | 1-4 周 | 2-8 周 | 8-24 周 | 30 天内 |
| **主要风险** | 没挖到真实痛点 | 选错了客户 | 没识别反对者 | 节奏被打乱 |
| **成交方式** | 四步问法链 | 小步承诺 | 多角色策略 | 1-7-3 时间节奏 |

## Strategic Selling 与 Clover 的融合

### 融合流程

```
Clover 1-7-3 节奏 + Strategic Selling 多角色策略

PRE 筛选 → 确认 ICP，识别决策复杂度
   ↓
G1 破冰 → 找到 Coach（第一个支持你的人）
   ↓
G2 诊断 → 通过 Coach 画决策流程地图
   +  Strategic Selling：问"还有谁会参与？"、"谁最后拍板？"
   ↓
G3 拉扯 → 识别 Champion 和 Competitor
   +  Strategic Selling：判断每个人对你的态度（+ / 0 / -）
   ↓
G4 承诺 → 培养 Champion
   +  Strategic Selling：给 Champion 内部演示的弹药
   ↓
G5 价值 → 准备见 Economic Buyer
   +  Strategic Selling：1 页 executive summary + ROI 数据
   ↓
G6 收口 → 管理决策流程
   +  Strategic Selling：定时间、定流程、定下一步
   ↓
G7 跟单 → 持续接触所有角色
   +  Strategic Selling：通过 Coach 监控内部态度变化
```

## 实战场景

### 场景 1：第一通电话后画决策流程地图

```
销售："张总，今天聊得很开心。我想问一下——
      这个项目除了你，还有谁会参与评估？"

客户："技术那边的李工也会看，最后王总拍板。"

销售："王总一般多久做一次决定？"

客户："要看情况，快的话 2 周，慢的话 1-2 个月。"

销售："明白。那李工对这种方案一般是什么态度？"

客户："他比较保守，之前用过类似的东西说效果一般。"

销售："好，我记下了。那我建议这样——
      下次演示我专门准备一个技术环节，
      直接解答李工的顾虑。你看可以吗？"

客户："可以。"

→ 销售在 5 分钟内获取了 3 个关键信息：
  1. 决策者：王总（EB）
  2. 技术评审：李工（潜在 Competitor）
  3. 决策周期：2 周-2 个月
```

### 场景 2：培养 Champion

```
销售（对 Coach）：
"张总，我准备了一下下周的演示。
这是 3 个可能被问到的问题和回应，
你看看能不能帮我在内部会议上用？"

Coach：
"好的，这个数据很有说服力。
我下周的周会上会提到。"

销售：
"太好了。如果李工问技术问题，
你可以说我们的系统通过了 3 个安全认证，
具体细节我可以直接回答。"

→ Champion 有了"弹药"，会在内部会议上帮你说
```

## Strategic Selling 快速检查清单

开始一个大订单前，问自己：

- [ ] 我定义了 Winning Result 吗？（金额、时间、范围）
- [ ] 我画了决策流程地图吗？
- [ ] 我确认了谁是 Economic Buyer 吗？
- [ ] 我有 Coach 吗？
- [ ] 我有 Champion 吗？
- [ ] 我识别了 Competitor 吗？
- [ ] 我知道每个人对我的态度吗？（+ / 0 / -）
- [ ] 我知道 Competitor 的真正顾虑吗？
- [ ] 我的 Champion 有内部弹药吗？
- [ ] 我准备好见 Economic Buyer 了吗？（ROI + 案例 + 摘要）
- [ ] 我管理了决策流程的节奏吗？

如果以上有 3 个以上的"否"，你的大订单正处于高风险中。
