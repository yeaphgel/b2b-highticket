# Gap Selling - 现代 B2B 成交法（帮客户看到问题）

**创始人**：马克·韦夏克（Mark Weigel） | **适用**：SaaS + 现代 B2B | **核心问题**：客户往往不知道自己有问题

---

## 核心洞察：大多数客户是"不知道自己不知道"

### Gap Selling 为什么必需？

**传统销售的假设**：
```
假设：客户已经知道自己有问题
现实：80% 的现代 B2B 买家不知道自己有问题

→ SPIN/Sandler 的问诊法对"不知道的客户"效果有限
→ 需要一个"激发意识"的方法
```

**Gap Selling 的假设**：
```
大多数客户处于"不知道他们的现状与理想状态有多大的差距"

销售的工作 = 帮他们"看到"这个 Gap
一旦客户看到了 Gap，他们会自己找解决方案
```

### 三个客户状态

| 客户状态 | 特征 | 销售方法 |
| --- | --- | --- |
| **不知道（Unaware）** | 完全不认识到自己有问题 | Gap Selling - 帮他们看到 Gap |
| **知道但不紧急（Aware）** | 知道有问题但没想改变 | SPIN - 问诊法放大问题紧迫性 |
| **已决定（Decided）** | 已经知道要买，在比较方案 | Sandler - 成交与异议处理 |

**现代 B2B 的现状**：70% 的前期接触都发生在"不知道"阶段

---

## Gap Selling 的 4 步流程

### 第 1 步：了解现状（As-Is）

**目标**：客户描述他们现在的状态

**关键问题**

```
"你们目前是怎么处理这一块的？"
"现在的系统运作得怎么样？"
"你们对现在的情况满意吗？"
"现在最大的问题是什么？"
```

**Gap Selling 的特殊之处**：
- 不是为了"诊断问题"（那是 SPIN 的目的）
- 而是为了"建立现状"作为基准

**实战例子**

```
销售："你们现在怎么管理销售管道的？"

客户："我们用 Excel，每周手动更新，
      销售经理每周花 3 小时整理数据。"

销售："明白。那这套流程运作多久了？"

客户："大概 3 年了。"

销售："那 3 年来，数据的准确度怎么样？能拿来做决策吗？"

客户："其实……经常不准，因为销售人员常常忘记及时更新。
      有时候看着数据说一套，实际销售流程又是另一套。"

销售："这个 Gap 对你们造成什么影响吗？"

客户："嗯，我们有时候会误判销售形势，
      导致财务预报经常出偏差。"

→ 现在客户开始意识到"现状"有问题
→ 但他们还不知道"理想状态"是什么
```

**与 Clover 的对应**：= Clover G2 诊断，但更关注"现状描述"本身

---

### 第 2 步：定义理想状态（To-Be）

**目标**：客户描述他们"理想中"应该是什么样

**关键问题**

```
"在理想情况下，这一块应该怎么运作？"
"如果你能重新设计这个流程，你会怎么做？"
"你觉得最高效的方式是什么？"
"如果时间和成本都不是问题，你想要什么？"
```

**Gap Selling 的神奇之处**：让客户自己描述理想，而非销售告诉他们

**实战例子**

```
销售："在理想情况下，销售管道应该怎么运作？"

客户："理想的话，数据应该是实时的。
      销售输入一条记录，管理层立即就能看到。
      不需要手动汇总，系统自动生成报表。"

销售："除了数据实时，还有什么？"

客户："还有预测准度。我们现在经常预测不准，
      导致财务部和销售部总是对不上账。
      理想的情况下，这个 Gap 应该小于 5%。"

销售："那如果预测准度提高到 95% 以上，
      对公司会有什么影响？"

客户："太大了。我们可以更准确地制定计划，
      财务可以更好地预估现金流，
      董事会也能对公司前景有信心。"

→ 现在客户已经清楚描述了理想状态
→ 他们自己意识到了"现状 vs 理想"的巨大差距
```

**与 Clover 的对应**：= Clover G3 向上拉扯（放大欲望），但用"理想状态"的形式

---

### 第 3 步：定量化 Gap（Quantify the Gap）

**目标**：用数字和实际影响来衡量 Gap 的大小

**为什么要量化**：
- "有点问题" vs "损失 ¥2000 万/年" → 紧迫性完全不同
- 数字让 Gap 变得"真实"而不是"抽象"

**量化 Gap 的维度**

| 维度 | 例子 | 数量化后的影响 |
| --- | --- | --- |
| **时间成本** | 销售每周手动更新数据 3 小时 | 月度 12 小时，年度 144 小时 = ¥20 万人力成本 |
| **错误成本** | 预测偏差导致财务预报错误 | 每个季度平均偏差 20%，意味着融资决策风险 ¥1000 万 |
| **机会成本** | 销售经理花 3 小时整理数据，无法做战略规划 | 无法做新市场开拓，机会损失 ¥500 万/年 |
| **风险成本** | 信息延迟导致不能及时调整策略 | 竞品可能抢先，市场份额风险 |

**实战例子**

```
销售："你刚才说销售每周花 3 小时更新数据。
      你们有多少销售？"

客户："大概 30 个。"

销售："那一周就是 30×3=90 小时，一月就是 360 小时。
      这 360 小时用平均工资算，一个月的成本大概多少？"

客户："大概 ¥5 万吧……哦天哪，一年就是 ¥60 万。"

销售："对。而且还有隐形成本——他们花时间做这个，
      就没时间去真正的销售活动。
      如果他们多 10% 的时间用来售卖，能带来多少额外收入？"

客户："每个销售平均月收入 ¥20 万，
      10% 的时间差不多是 ¥2 万/人/月，
      30 个人就是 ¥60 万/月，¥720 万/年。"

销售："所以，这个 Gap 对你们的年度影响是多少？"

客户："直接成本 ¥60 万，机会成本 ¥720 万……
      总共大概 ¥780 万/年。"

销售："现在理解为什么这个问题值得解决了吗？"

客户："理解了。但是我从来没想过数字是这么大……"

→ 从"有点问题"变成"一年亏 ¥780 万"
→ 现在客户开始主动思考"怎么解决"
```

**Gap Selling 的威力**：不是销售说"我们能帮你省 ¥780 万"，而是客户自己算出来

**与 Clover 的对应**：= Clover G3 极限拉扯 + G5 价值展示的综合版

---

### 第 4 步：呈现视角（Present the Vision）

**目标**：呈现"实现理想状态"的新视角或方法

**Gap Selling 的关键区别**：
- 不是推销产品，而是"展示新的可能性"
- 让客户意识到有这样的方案存在

**实战例子**

```
销售："现在你看到了这个 Gap 的规模——
      一年 ¥780 万的影响。
      
      我想给你看一个不同的视角。
      
      在我们合作的企业里，有家和你们情况类似的公司。
      他们以前也是用 Excel，后来换了一个系统，
      自动化了数据收集和报表生成。
      
      结果怎么样呢？
      1. 数据准度从 75% 提升到 98%
      2. 销售每周省掉 3 小时的行政工作
      3. 财务预报偏差从 20% 降到 3%
      
      这个新系统改变了他们对销售流程的看法。
      他们甚至发现了原来没看到的销售机会。
      
      你们想不想看看他们是怎么做的？"

客户："想！这正是我们需要的。"

→ 注意：销售没有说"我们的产品有什么功能"
→ 而是说"这里有一个新的可能性，看起来很有效"
→ 客户现在主动想了解
```

**与 Clover 的对应**：= Clover G5 价值展示，但 Gap Selling 更强调"新视角"而非"产品功能"

## Gap Selling 的 3 个关键概念

### 1. The Vision（理想状态的愿景）

不是产品描述，而是客户的理想未来：

```
错误：
"我们的系统有 AI 预测、自动化报表、CRM 集成……"

正确（Gap Selling 方式）：
"在我们合作的企业里，销售管理变成了这样：
- 销售输入一条记录，管理层立即看到
- 系统自动预测下月收入，准度 95% 以上
- 没有人再需要手动汇总数据
- 销售经理每周多出 3 小时做战略性工作
- 公司对收入预测有信心，能更好地制定计划"

→ 这是一个"新现实"，而非"产品特性"
```

### 2. The Story（故事，而非案例）

Gap Selling 强调用故事（而非数据）来展示 Vision：

```
"有家餐饮连锁，30 家店铺分散在全国。
他们原来也是中央统计报表，每周花 2 天。
后来用了一个系统，店铺数据自动汇总。

有意思的是，他们发现某个店的销售模式特别高效。
原来他们想复制到其他店，但因为数据不清楚，一直没做。
现在数据清楚了，他们复制了这个模式，
结果整个连锁的坪效提升了 15%。

这个故事说明：当数据清楚了，你能看到原本看不到的机会。"

→ 故事比"我们的系统提升准度 20%"更有说服力
```

### 3. The Proof（证明视角有效）

不是"我们的产品好"，而是"这个新视角确实有效"：

```
错误：
"我们的产品获得了五星评价，客户满意度 98%……"

正确（Gap Selling 方式）：
"和你们情况类似的 5 家企业，都采用了这个新视角。
结果怎么样？
- 数据准度都提升到 95% 以上
- 销售人员都省掉了行政工作时间
- 财务预报偏差都降到 5% 以内
- 他们都发现了原本看不到的业务机会

这证明，当你改变了对销售流程的看法，
整个企业都能受益。"

→ 证明的是"视角的有效性"，不是"产品的有效性"
```

## Gap Selling vs SPIN vs Sandler vs Clover

| 维度 | SPIN | Sandler | Gap Selling | Clover |
| --- | --- | --- | --- | --- |
| **客户前提** | 已知问题，需要诊断 | 有预算和权力，需要评估 | 不知道问题，需要唤醒 | 多维度，需要时间+诊断+行动 |
| **销售角色** | 诊断医生 | 顾问/审核官 | 愿景引导者 | 销售力杠杆 |
| **成交前提** | 客户自己说出价值 | 客户自己说出条件 | 客户自己看到 Gap 有多大 | 客户经历 1-7-3 全流程 |
| **时间框架** | 单次对话（30-60min） | 可多次互动（筛选客户） | 多次深度对话（唤醒意识） | 1 天痛点锚定 + 7 天信任 + 30 天成交 |
| **核心问题** | "你有什么问题？" | "你值不值得我帮？" | "你看到你的 Gap 有多大吗？" | "1 天内锚定痛点，7 天建信任，30 天内成交" |
| **适用场景** | 高价值 ToB（£50K+）| 中高价值需筛选 | SaaS/现代 ToB（未意识客户） | 全 B2B 场景，特别是中国市场 |

## Gap Selling 与 Clover 的完整融合

### 融合点分析

**Clover G2 诊断 → Gap Selling 的现状描述**
- Clover：用问题库让客户说出痛点
- Gap Selling：进一步让客户描述"完整的现状图景"

**Clover G3 极限拉扯 → Gap Selling 的 Gap 量化**
- Clover：向下放大痛苦，向上放大欲望
- Gap Selling：用数字量化这个 Gap 的真实影响（¥）

**Clover G5 价值展示 → Gap Selling 的愿景呈现**
- Clover：用案例和专家身份展示价值
- Gap Selling：用故事和新视角展示"达成理想状态的可能性"

### 融合流程

```
Clover 的整个对话流程 + Gap Selling 的"看到 Gap"逻辑

G1 破冰 
   ↓
G2 诊断（让客户描述现状）
   + Gap Selling 元素：不仅问"有什么问题"
   + 而是问"现在怎么运作的"、"这样持续多久了"、"对业务有什么影响"
   ↓
G3 极限拉扯（放大 Gap）
   + Gap Selling 元素：除了情绪化的拉扯（向下痛苦、向上欲望）
   + 加入数字化的 Gap 量化（¥780 万/年 的影响）
   ↓
G4 承诺
   ↓
G5 价值展示（呈现解决方案）
   + Gap Selling 元素：不是展示"产品有什么功能"
   + 而是展示"达成这个理想状态的新视角"（故事 + 证明）
   ↓
G6 收口（成交）
```

## Gap Selling 的实战话术

### 第 1 步：了解现状（As-Is）

```
"你们现在是怎么处理销售管道管理的？"

"这套系统已经用多久了？"

"现在运作得怎么样？对业务有什么影响吗？"

"如果要改变，最大的障碍是什么？"
```

### 第 2 步：定义理想状态（To-Be）

```
"在理想的世界里，销售管道应该怎么运作？"

"如果时间和成本都不是问题，你最想要什么样的方式？"

"如果能实现这样的理想状态，对公司最大的好处是什么？"

"用数字说，这会带来多少收益？"
```

### 第 3 步：量化 Gap

```
"你说销售每周花 3 小时更新数据。
多少销售人员？[客户答：30 个]

那一年就是 30×3×52 = 4680 小时。
按平均工资，一年成本大概多少？"

"除了直接成本，还有什么隐形成本吗？
比如，他们没时间做的事……"

"综合算下来，这个 Gap 对公司一年的影响是多少？"
```

### 第 4 步：呈现愿景

```
"我想给你看一个不同的可能性。

有家和你们很像的公司，也遇到过同样的问题。
后来他们改变了对这个流程的看法……
[讲故事]

结果是什么呢？
[呈现愿景 + 证明]

你觉得这个方向对你们有帮助吗？"
```

## Gap Selling 的心理学基础

### 为什么 Gap Selling 这么有效？

**1. 对比效应（Contrast Effect）**

人脑不善于评估绝对值，但擅长比较。

Gap Selling 强制客户比较"现状 vs 理想"，这让 Gap 变得超级明显。

```
不用 Gap Selling：
客户心想："这个系统好像还不错……"

用 Gap Selling：
客户心想："现在 ¥60 万人力成本，理想中 0 成本……
            现在 75% 准度，理想中 98% 准度……
            这个 Gap 太大了！必须解决！"
```

**2. 所有权效应（Ownership Effect）**

人们对"自己想出来的"比"别人告诉他的"更相信。

Gap Selling 让客户自己描述理想状态、自己算出 Gap 的大小。

```
不用 Gap Selling：销售说"我们能帮你省 ¥780 万" → 客户很懀疑

用 Gap Selling：客户自己算出"这个 Gap 是 ¥780 万" → 客户深信不疑
```

**3. 现状偏差（Status Quo Bias）**

人天生抗拒改变。但如果 Gap 足够大，改变的欲望就会超过抗拒。

```
小 Gap："还好吧，继续用 Excel 也可以。"

大 Gap（¥780 万）："天哪，这必须改！"
```

## Gap Selling 快速检查清单

使用 Gap Selling 之前，问自己：

- [ ] 我的目标客户是否在"不知道他们有问题"的阶段？
- [ ] 他们现在正在用的方案是什么？（现状）
- [ ] 理想的方案应该是什么？（愿景）
- [ ] 这个 Gap 可以量化吗？（数字）
- [ ] 我是否有案例或故事来证明这个新视角有效？（证明）
- [ ] 客户是否会因为看到 Gap 而自动想要解决？

如果这些都 Yes，那么 Gap Selling 就很适合。
