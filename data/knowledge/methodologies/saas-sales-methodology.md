# SaaS 销售方法论

**核心理论**：Winning by Design《The SaaS Sales Method》+ SaaS Sales Academy | **适用**：SaaS / 订阅制产品 | **核心问题**：SaaS 销售不只是签单，而是 LTV（客户生命周期价值）最大化

---

## 核心洞察：SaaS 销售的逻辑和传统销售完全不同

### SaaS 销售 vs 传统销售

```
传统销售：
目标：签单 → 结束
衡量：单笔金额
客户生命周期：一次性

SaaS 销售：
目标：签单 → 激活 → 留存 → 扩展 → 续约
衡量：LTV（客户生命周期价值）+ MRR（月度经常性收入）
客户生命周期：持续 3-5 年
```

### SaaS 销售的 3 个核心指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **CAC** | 获客成本 = 销售+营销费用 / 新客户数 | 获取一个客户要花多少钱 |
| **LTV** | 客户生命周期价值 = 月费 × 平均留存月数 | 一个客户值多少钱 |
| **LTV:CAC** | 理想比例 ≥ 3:1 | 每花 ¥1 获取客户，回报 ≥ ¥3 |

**为什么这很重要**：
```
传统销售：成交 = 成功
SaaS 销售：成交 = 开始

如果客户签了单但 30 天后流失：
→ CAC 花了 ¥5000
→ 只收到 ¥1000
→ 亏损 ¥4000

所以 SaaS 销售的重点不是"签更多单"，而是
"签对的单"（那些会留存、会扩展的客户）
```

## SaaS 销售的 5 步流程

### 第 1 步：Qualification（资格审查）—— 找到对的客户

**SaaS 资格审查的 ICP 模型**：

```
ICP = Ideal Customer Profile（理想客户画像）

不是"所有需要内容运营的公司"
而是：
- 行业：餐饮/零售/电商（我们最擅长的）
- 规模：年营收 ¥500 万 - ¥5000 万
- 痛点：内容产出不稳定，获客成本上升
- 预算：愿意为结果付费，不只是工具
- 决策：老板直接参与决策
```

**Sandler + SaaS 的资格审查**：
```
5 个维度都要符合：
1. 预算：有 ¥5K+/月 的营销预算
2. 痛点：内容运营是明确的痛点
3. 决策：老板/负责人能当场做决定
4. 时间：30 天内有启动意愿
5. 匹配：行业和规模在我们的能力范围内
```

**不合格客户的信号**：
```
- "我们就想先试试"（没有预算意识）
- "让员工先做做看"（不是决策者）
- "你们能不能先免费试用 3 个月"（价值不匹配）
- "我们还在考虑要不要做内容"（没有痛点）
```

---

### 第 2 步：Discovery（发现）—— 理解客户的业务目标

**SaaS 销售的 Discovery 不是问"你有什么需求"**：

```
普通销售的 Discovery：
"你们现在用什么工具？"
"你们有什么需求？"
→ 客户说"我想要 XX 功能" → 你推销功能

SaaS 销售的 Discovery：
"你们今年的增长目标是什么？"
"获客渠道中，内容占多少比例？"
"如果内容这块解决了，对你们的业务意味着什么？"
→ 客户说出"业务目标" → 你连接产品价值
```

**Discovery 的黄金问题库**：

```
业务层：
1. 你们今年的业务增长目标是多少？
2. 目前获客成本是多少？目标是多少？
3. 内容获客在你们的整体策略中占什么位置？

团队层：
4. 现在谁在负责内容？几个人？
5. 内容团队的能力怎么样？
6. 如果内容这块一直这样，对团队有什么影响？

个人层：
7. 内容运营对你的 KPI 影响大吗？
8. 你个人最想从这个事情中解放出来的是什么？
9. 如果内容这块做得特别好，对你个人意味着什么？
```

**与 Clover 的对应**：= G2 诊断的深化版，但 SaaS 销售强调要"从业务目标出发"，而不只是"从痛点出发"

---

### 第 3 步：Demo（演示）—— 展示价值而非功能

**SaaS 演示的黄金法则**：
```
不要演示功能，演示结果。

错误：
"我们的系统有 AI 内容生成、自动排期、多平台分发、
数据分析、团队协作……"
→ 客户觉得"功能很多"，但不知道"这对我有什么用"

正确：
"你们的团队每周花 20 小时在内容上。
用我们的方案后，这个时间会变成 2 小时——
省下的 18 小时，你可以用来做战略规划。
我来给你看一个和你情况类似的客户的使用方式。"
→ 客户看到了"结果"，而不是"功能列表"
```

**Demo 的 3 步结构**：
```
1. 连接 Discovery（"上次你提到…"）
   "上次你说到内容产出效率是最大瓶颈。
    今天我想给你看我们怎么帮客户解决这个问题。"

2. 展示结果（不是功能）
   "你看——这是客户 A 的 dashboard。
    他们的内容产出从每周 3 条提升到 14 条，
    但团队投入时间从 20 小时降到 2 小时。
    这就是我们要帮你做到的。"

3. 验证匹配（"你觉得呢？"）
   "这个方向跟你们的需求匹配吗？
    有没有什么觉得不太对的？"
```

**Demo 中的 Closer's Move**：
```
"你看，这就是我们怎么帮客户解决问题的。
但最重要的是——你觉得这能帮你解决你刚才说的问题吗？"
→ 让客户自己说出"是"
→ 这个"是"比任何功能演示都有力量
```

### 第 4 步：Trial（试用）—— 降低风险，加速决策

**SaaS 试用的设计原则**：
```
试用期不是"让客户随便用"
而是"有引导的价值体验"
```

**SaaS 试用流程**：
```
Day 1：Onboarding（上手引导）
  - 设置客户的目标（"30 天内你要达成什么？"）
  - 完成初始配置
  - 安排 1 对 1 培训

Week 1：Early Win（早期胜利）
  - 帮客户完成第一次内容产出
  - 展示数据（"你看，已经产出了 5 条内容"）
  - 客户感受到价值

Week 2-3：Value Expansion（价值扩展）
  - 引导使用更多功能
  - 展示更多结果
  - 客户越来越依赖产品

Week 4：Conversion（转化）
  - 回顾 30 天的结果
  - 连接商业价值（"30 天带来了 X 个线索"）
  - 讨论续约和升级
```

**试用成功的指标**：
```
激活指标：客户完成了核心操作（第一次内容产出）
留存指标：客户每周都在使用（使用频率）
扩展指标：客户主动探索了更多功能
转化指标：客户主动询问价格和续约
```

---

### 第 5 步：Close & Onboard（成交 + 上手）

**SaaS 成交的心法**：
```
传统销售成交：签单 → 结束
SaaS 销售成交：签单 → 开始

所以 SaaS 成交不只是"签单"，
而是"帮客户做好成功的准备"
```

**成交时的 Onboarding 承诺**：
```
"恭喜你加入！接下来我们会这样合作：
- Day 1：我们帮你完成初始设置，30 分钟培训
- Week 1：帮你产出第一批内容，确保上手顺利
- Week 2-4：每周一次检查，确保你获得预期结果
- 之后：你的专属客户成功经理会持续支持你

你不需要担心'不会用'——我们会全程指导。"
→ 客户买的不只是产品，还有"成功保障"
```

## SaaS 销售的客户成功（Customer Success）

### 为什么 Customer Success 是 SaaS 销售的核心？

```
SaaS 商业模式的核心：
收入 = 月费 × 客户数 × 平均留存月数

如果客户流失率高：
→ 每个月都需要找新客户填补
→ CAC 越来越高
→ 利润越来越低

如果客户留存率高：
→ 老客户持续付费
→ 老客户还会升级（扩展收入）
→ CAC 摊薄，利润增加
```

### 客户成功的 3 个阶段

| 阶段 | 目标 | 关键动作 |
|------|------|---------|
| **激活（0-30 天）** | 客户第一次感受到价值 | 上手培训、早期胜利、快速反馈 |
| **留存（30-180 天）** | 客户养成使用习惯 | 定期检查、数据报告、主动支持 |
| **扩展（180 天+）** | 客户购买更多/更贵 | 升级方案、新增功能、推荐新客户 |

### 客户健康度指标

```
活跃指标：
- 每周登录次数
- 每周内容产出数量
- 功能使用深度

风险指标：
- 登录频率下降
- 内容产出减少
- 客服工单增加
- 合同即将到期

健康客户特征：
- 每周登录 3+ 次
- 每周产出 5+ 条内容
- 主动探索新功能
- 给正面反馈
```

## SaaS 销售 vs 传统销售

| 维度 | 传统销售 | SaaS 销售 |
|------|---------|----------|
| **销售目标** | 签单笔数/金额 | LTV + MRR |
| **销售周期** | 一次性 | 持续（3-5 年） |
| **成交后** | 结束 | 开始（Onboarding → Success → Expansion） |
| **关键角色** | 销售代表 | 销售 + 客户成功经理 |
| **成功指标** | 签单金额 | 留存率、扩展率、NPS |
| **客户流失** | 不影响 | 致命（直接减少收入） |
| **销售心态** | "签更多单" | "签对的单" |

## SaaS 销售与 Clover 的融合

```
Clover 1-7-3 + SaaS 销售方法

PRE 筛选 → SaaS ICP：确认客户符合 SaaS 理想画像
   ↓
G1 破冰 → 展示 SaaS 的价值认知
   +  "我们帮 500+ 企业把内容产出提升了 5 倍"
   ↓
G2 诊断 → SaaS Discovery：从业务目标出发
   +  "你们今年的增长目标是什么？"
   +  Clover 黄金问题库 + SaaS 业务层/团队层/个人层
   ↓
G3 拉扯 → 量化 SaaS 的价值差距
   +  "现在 20 小时/周 vs 理想 2 小时/周"
   +  "一年 ¥780 万人力成本 vs 年费 ¥6 万"
   ↓
G4 承诺 → 引导试用决策
   +  "不如先试用 30 天，看看效果？"
   ↓
G5 价值 → SaaS Demo：展示结果而非功能
   +  "这是和你类似的客户，30 天后的结果"
   ↓
G6 收口 → 试用转化
   +  "试用数据证明有效，我们正式签约？"
   ↓
G7 跟单 → SaaS Customer Success
   +  Day 1 Onboarding → Week 1 Early Win
   +  Week 2-4 Value Expansion
   +  30 天 Conversion
```

## SaaS 销售快速检查清单

每次 SaaS 销售对话后，问自己：

- [ ] **Qualification**：客户符合我们的 ICP 吗？（行业、规模、预算、决策权）
- [ ] **Discovery**：我了解客户的业务目标了吗？（不只是痛点）
- [ ] **Demo**：我展示了结果而不是功能吗？
- [ ] **Trial**：试用有明确的目标和时间表吗？
- [ ] **Onboarding**：成交后的上手流程准备好了吗？
- [ ] **Success**：我知道客户的健康度指标是什么吗？

如果以上有 2 个以上的"否"，你的 SaaS 销售转化率会低于行业平均水平。
