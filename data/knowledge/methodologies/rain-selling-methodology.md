# RAIN Selling - 现代关系销售法

**创始人**：基普林格 & 帕拉尼普（RAIN Group） | **适用**：现代 B2B 销售（所有规模） | **核心**：建立信任 → 培养信任 → 维持信任

---

## 核心洞察：现代 B2B 买家已经变了

### 为什么传统销售方法效果在下降？

**现代买家的变化**：
```
2010年：买家需要销售提供信息
2026年：买家已经通过网上搜索了解了 70%
```

**结果**：
```
"推销"不再有效——买家不需要你告诉他产品有什么
"关系"变得最关键——买家需要的是"值得信任的顾问"
```

**RAIN 的研究发现**（对 500+ B2B 买家的调研）：
```
买家选择供应商的 3 大因素：
1. 销售人员是否理解我的需求（78%）
2. 是否带来了新的洞察（65%）
3. 是否值得信赖（62%）

"价格"只排在第 5 位（47%）
```

## RAIN 四字母的含义

| 字母 | 原文 | 核心含义 |
|------|------|---------|
| **R** | Relationship | 建立关系——买家愿意跟你说话 |
| **A** | Ability | 展示能力——买家相信你做得好 |
| **I** | Integrity | 保持诚信——买家相信你不会骗他 |
| **N** | Nurture | 长期培养——买家觉得你值得长期合作 |

## RAIN 的 5 步销售流程

### 第 1 步：Research（研究）—— 在接触前

**目标**：了解买家比买家自己想的还多

**RAIN 的差异化**：不是"了解客户的基本信息"，而是"了解客户的业务、行业、竞争对手"

**研究清单**：
```
- [ ] 客户公司的最近新闻/融资/战略变化
- [ ] 客户所在行业的趋势和挑战
- [ ] 客户竞争对手的动向
- [ ] 关键决策人的 LinkedIn 动态（最近关注什么）
- [ ] 客户公司的产品/服务/商业模式
```

**实战话术（第一通电话）**：
```
"王总，我看了你们上周发布的 Q3 财报，
提到新市场的拓展是你们今年的重点。
我们在服务一家类似规模的公司时，
帮助他们在新市场 3 个月就拿到了第一批客户。
你 10 分钟后有空聊 5 分钟吗？"

→ 这不是"你好，我是 XX 公司的..."的推销
→ 这是"我研究了你的业务，我有一个有价值的洞察"的开场
```

**与 Clover 的对应**：= G1 破冰的"有备而来"，但 RAIN 强调研究要深入到"行业层面"而不只是"账号层面"

---

### 第 2 步：Reach Out（接触）—— 创造有价值的对话

**RAIN 的核心原则**：每次接触都要给客户带来价值

**无效的接触**：
```
"你好，我是 XX 公司的，想给你介绍一下我们的服务。"
→ 客户想："又一个推销的"
```

**RAIN 方式的接触**：
```
"王总，我最近写了一份《电商行业内容营销趋势报告》，
里面有你们行业 3 个值得关注的变化。
想分享给你，看看有没有启发。"
→ 客户想："这个人有研究，有价值"
```

**接触的价值阶梯**：
```
最低价值：介绍自己的公司/产品
中等价值：分享行业趋势/数据
最高价值：分享针对客户公司的具体洞察
```

**与 Clover 的对应**：= Clover 互惠原则的体现——G1 破冰前先给价值（竞品分析、行业报告）

---

### 第 3 步：Reveal（揭示）—— 对话中展示洞察

**目标**：让客户"看到"他们之前没看到的东西

**RAIN 的核心差异化**：
```
普通销售：客户说问题 → 销售说方案
RAIN 销售：客户说问题 → 销售带来新视角 → 客户看到更大的问题

例子：
客户："我们的内容产出效率太低了。"
普通销售："我们的系统可以自动化内容生产..."
RAIN 销售："效率低只是表象。
        我研究了你们 30 篇内容，发现更大的问题是：
        内容的方向和目标受众不匹配。
        你们在跟工程师说话，但客户是老板。
        这个问题不改，产出再快也没用。"
```

**揭示新视角的 3 步**：
```
1. 确认你理解了客户的现状
   "你刚才说...我理解对吗？"

2. 引入新信息/数据/案例
   "我在研究你们的行业时发现..."

3. 展示这个新视角的影响
   "如果这个假设是对的，那意味着..."
```

**与 Clover 的对应**：= G2 诊断 + G3 拉扯的深化版。Clover 用提问引导客户发现，RAIN 用洞察直接带来新视角

---

### 第 4 步：Respond（回应）—— 处理异议和疑虑

**RAIN 的异议处理框架**：

```
不是"克服异议"，而是"理解 + 回应"

客户："价格太贵了。"

普通销售："但我们的质量更好..."
→ 陷入价格辩论

RAIN 销售："我理解你的顾虑。能问一下——
            你是觉得这个投资不值这个价？
            还是你们现在有预算限制？"
→ 探索真正的原因
```

**RAIN 的异议分类**：
```
1. 信息不足型："我不确定你们能不能做到..."
   → 回应：提供更多证据（案例、数据、试用）

2. 风险担忧型："之前用过类似的东西，效果不好..."
   → 回应：承认风险 + 提供保障（试用期、退款承诺）

3. 利益不匹配型："这个功能我们不需要..."
   → 回应：调整方案，聚焦客户真正需要的

4. 决策流程型："我要回去跟团队讨论..."
   → 回应：主动了解决策流程，提供内部演示支持
```

**与 Clover 的对应**：= G6 异议处理，但 RAIN 强调"先理解再回应"，而非 Sandler 的"反向"

---

### 第 5 步：Nurture（培养）—— 长期关系建设

**RAIN 的研究发现**：
```
58% 的买家选择供应商是因为"销售人员持续提供价值"
即使这个供应商不是最便宜、功能最多的
```

**Nurture 的核心**：即使在没成交的时候，也要持续提供价值

**Nurture 节奏**：
```
Week 1：感谢 + 诊断价值（已成交/未成交都送）
Week 3：行业触点（新的数据/趋势分享）
Week 5：案例证明（同行业成功案例）
Week 8：个人化关注（客户的 LinkedIn 动态回应）
Week 12：价值回顾（上次聊天后的进展更新）
Month 4+：持续提供行业洞察（每月 1 次）
```

**Nurture 的关键原则**：
```
1. 不要只"催单"时才出现
2. 每次联系都要有具体的价值（文章、数据、案例）
3. 不要"广撒网"——信息要针对具体客户
4. 要"记得"——记住上次对话中的细节
```

**Nurture 话术示例**：
```
"张总，上周你提到你们在考虑 Q4 的战略调整。
我刚好看到一份报告，讲你们行业的 Q4 趋势。
里面有几个数据点你可能感兴趣，分享给你。"

→ 没有"催单"
→ 有"具体价值"
→ 客户会觉得"这个人记得我说的话"
```

**与 Clover 的对应**：= G7 跟单系统的深化版。Clover 有明确的 D1-D30 节奏，RAIN 的 Nurture 是更长期的（30 天→90 天→180 天）

## RAIN 的信任方程式

```
信任 = 诚信 + 能力 + 一致性

诚信（Integrity）：客户相信你不会骗他
→ 不夸大、不承诺做不到的事、承认产品局限

能力（Ability）：客户相信你做得好
→ 展示专业度、案例、数据、行业认知

一致性（Consistency）：客户相信你会持续做到
→ 每次承诺都兑现、持续提供价值、长期跟进
```

**信任建立的节奏**：
```
第一次见面：诚信（你看起来靠谱）
第二次见面：能力（你带来有价值的洞察）
第三次见面：一致性（你兑现了之前的承诺）
之后：信任已经建立，客户开始主动找你
```

## RAIN 的买家行为洞察

### 买家最看重的销售行为（RAIN 调研数据）

| 行为 | 买家重视度 | 销售经常做的比例 |
|------|-----------|----------------|
| 倾听多于说话 | 89% | 42% |
| 了解我的行业 | 85% | 28% |
| 带来新的想法 | 78% | 23% |
| 理解我的需求 | 76% | 56% |
| 提供数据支持 | 65% | 38% |
| 及时回应 | 58% | 71% |

**差距最大的**："了解我的行业"（85% vs 28%）和"带来新的想法"（78% vs 23%）

→ 这说明：大多数销售在"展示能力"和"带来洞察"上做得远远不够
→ 这也正是 RAIN 方法的核心差异化

## RAIN vs SPIN vs Sandler vs Clover

| 维度 | SPIN | Sandler | RAIN | Clover |
|------|------|---------|------|--------|
| **核心哲学** | 诊断客户需求 | 评估客户资格 | 建立信任 | 1-7-3 时间节奏 |
| **权力动力** | 平等顾问 | 销售主导 | 平等合作 | 销售主导节奏 |
| **差异化** | 问诊法 | 反向销售 | 洞察 + 持续价值 | 系统化跟进 |
| **时间框架** | 单次对话 | 多次互动 | 长期关系（90+ 天） | 30 天闭环 + 长期培育 |
| **适用场景** | 高价值 ToB | 需要筛选的订单 | 所有 B2B 订单 | 全 B2B 场景 |
| **关键优势** | 让客户自己说出痛点 | 筛选不合适的客户 | 买家最认可的信任建设 | 系统化执行力 |

## RAIN 与 Clover 的完整融合

### 融合流程

```
Clover 1-7-3 + RAIN 信任建设

PRE 筛选 → RAIN Research：深度研究客户行业和业务
   ↓
G1 破冰 → RAIN Reach Out：带着价值洞察接触客户
   +  RAIN：不是"介绍公司"而是"带来行业洞察"
   ↓
G2 诊断 → RAIN 倾听：80% 倾听，20% 提问
   +  RAIN：用 Clover 黄金问题库，但先让客户多说
   ↓
G3 拉扯 → RAIN Reveal：带来客户没看到的新视角
   +  RAIN：不是 Clover 的"放大痛苦"，而是"展示更大的格局"
   ↓
G4 承诺 → RAIN Integrity：诚信回应客户的疑虑
   +  RAIN：承认产品局限，不夸大承诺
   ↓
G5 价值 → RAIN Ability：用数据和案例展示能力
   +  RAIN：不只讲成功案例，讲"对客户行业的研究成果"
   ↓
G6 收口 → RAIN Respond：理解 + 回应异议
   +  RAIN：探索异议的真正原因，而非直接反驳
   ↓

   +  RAIN：从 Clover 的 D1-D30 扩展到 90+ 天的持续 Nurture
```

## RAIN 实战场景

### 场景 1：第一通电话的 RAIN 方式

```
销售："张总，感谢你抽时间。
      在打电话之前，我花了一些时间研究你们的业务。
      我发现了一些有意思的事——
      你们最近在新市场有投入，对吗？"

客户："对，我们在考虑东南亚。"

销售："有意思。我服务过 3 家做东南亚市场的公司，
      他们都有一个共同的发现：
      东南亚的内容营销节奏和国内完全不同。
      国内是一周一发，东南亚是一月一发都很难坚持。
      你们现在有研究过这个差距吗？"

客户："我们还没考虑到这点……"

销售："这就是我打电话的原因。
      我觉得这个问题很关键，但大多数公司没意识到。
      你 20 分钟后有时间吗？我想分享一些我们的研究。"

→ RAIN 的方式：研究 → 洞察 → 价值 → 对话
→ 不是推销，是"带来有用的信息"
```

### 场景 2：长期 Nurture（第 60 天）

```
销售（邮件/微信）：
"王总，你好。

上个月你提到你们在做年度战略规划，
我最近看到一份麦肯锡的报告，讲你们行业的
2027 年增长机会——里面有一些数据
可能对你做规划有帮助。

这是报告链接（摘要版），完整版我可以发你。

另外，上次提到的那家同行业公司，
他们 Q4 刚做完内容转型，数据很好。
你感兴趣的话，我可以安排你跟他们聊 15 分钟。

祝好，
XX"

→ 没有"催单"
→ 有 2 个具体的价值点（报告 + 客户对接）
→ 客户即使还没决定，也会觉得这个人很靠谱
```

## RAIN 快速检查清单

每次客户接触后，问自己：

- [ ] **Research**：我研究了客户的业务、行业、竞争对手吗？
- [ ] **Reach Out**：我带着有价值的信息去接触，而不是"推销"吗？
- [ ] **Reveal**：我带来了客户之前没看到的新视角吗？
- [ ] **Respond**：我"理解 + 回应"异议，还是直接反驳了？
- [ ] **Nurture**：我在非催单的时候也在持续提供价值吗？

如果以上有 2 个以上的"否"，你正在失去买家的信任。
